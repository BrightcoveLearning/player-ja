プレリリース広告プラグインのテスト

{{ page.description }}

概要

Brightcove Player に動画広告を表示することは、複雑な作業になることがあります。広告配信の成功には、複数のエンティティ（Brightcove Player 自体、広告サーバー、および特定の広告プラグイン）がすべて関与しています。IMA3 および FreeWheel プラグインの更新のテスト環境を提供するため、ブライトコーブはプラグインのプレリリースバージョンを入手します。このドキュメントでは、これらのプレリリースでテストする方法について説明します。

プレリリースは、ブライトコーブ Web プレーヤー開発者フォーラム、および IMA3 プラグインによる広告の更新履歴セクションで発表されます。 FreeWheelプラグインのドキュメントで広告を掲載しています。

スタジオでのIMA3のテスト

次の手順は、Studio でプレリリースの IMA3 プラグインをテストする方法を示しています。

  1. 安全のため、現在使用しているプラグインの現在のバージョン/プラグイン/プラグインバージョン参照.htmls を記録する必要があります。これにより、テスト前に使用していたプラグインのバージョンを復元できます。プラグインの最新バージョンはプラグインバージョンリファレンスのドキュメントでいつでも見つけることができます。
  2. を開きますプレイヤーモジュールを作成し、新しいプレーヤーを作成するか、プラグインを追加するプレーヤーを見つけます。
  3. プレーヤーのリンクをクリックして、プレイヤーのプロパティを開きます。
  4. 左側のナビゲーションメニューで [ 広告 ] をクリックします。
  5. [ クライアント側を有効にする (IMA)] のチェックを外します。すぐに手動でロードするバージョンに干渉する可能性があるため、この機能を有効にしないでください。
  6. [ 保存] をクリックします。
  7. 左側のナビゲーションメニューで [ プラグイン ] をクリックします。
  8. 次に、[ プラグインを追加] をクリックします。
  9. [ プラグイン名 ] に、と入力しますima3。
  10. JavaScript URL には、プレリリースの JavaScript ファイルへの URL を入力します。
  11. CSS の URL には、プレリリース CSS ファイルへの URL を入力します。
  12. [ オプション (JSON)]テキストボックスに構成オプションを入力します。 Plugins セクションを使用して IMA3 プラグインを設定したことがある場合は、ほとんどの場合、オプションを変更する必要はありません。
  13. [ 保存] をクリックします。
  14. プレーヤーを使用するには、[ 公開して埋め込む] をクリックします。プレビュープレーヤーを使用してテストするか、公開済みプレーヤーを公開して使用できます。。
  15. 開くダイアログを閉じるには、[ 閉じる] をクリックします。

スタジオでFreeWheelをテストする

次の手順は、Studio でプレリリースの FreeWheel プラグインをテストする方法を示しています。

  1. 安全のために、現在使用しているプラグインの現在のバージョンを記録する必要があります。これにより、テスト前に使用していたプラグインのバージョンを復元できます。プラグインの最新バージョンはプラグインバージョンリファレンスのドキュメントでいつでも見つけることができます。
  2. を開きますプレイヤーモジュールを作成し、新しいプレーヤーを作成するか、プラグインを追加するプレーヤーを見つけます。
  3. プレーヤーのリンクをクリックして、プレイヤーのプロパティを開きます。
  4. 左側のナビゲーションメニューで [ プラグイン ] をクリックします。
  5. 次に、[ プラグインを追加] をクリックします。プラグインのリストに FreeWheelPlugin が表示されている場合は、それをクリックします。
  6. [ プラグイン名 ] に、と入力しますFreeWheelPlugin。
  7. JavaScript の URL には、プレリリースの JavaScript ファイルへの URL を入力します。
  8. CSS の URL には、プレリリース CSS ファイルへの URL を入力します。
  9. [ オプション (JSON)] テキストボックスに構成オプションを入力します。[ プラグイン ] セクションを使用して FreeWheel プラグインを設定したことがある場合は、ほとんどの場合、オプションを変更する必要はありません。
  10. [ 保存] をクリックします。
  11. プレーヤーを使用するには、[ 公開して埋め込む] をクリックします。プレビュープレーヤーを使用してテストするか、公開済みプレーヤーを公開して使用できます。。
  12. 開くダイアログを閉じるには、[ 閉じる] をクリックします。

プログラムによる実装のテスト

プログラムによってプラグインを実装した場所のテストは、IMA3 プラグインでも FreeWheel プラグインを使用している場合でも、同じです。基本的には、JavaScript および CSS ファイルへの既存のパスをプレリリースバージョンのパスに置き換えるだけです。このタスクを実行する手順は次のとおりです。

  1. 安全のため、現在使用しているプラグインの現在のバージョンを記録する必要があります。これにより、テスト前に使用していたプラグインのバージョンを復元できます。プラグインの最新バージョンは、「プラグインバージョンリファレンス」ドキュメントでいつでも見つけることができます。
  2. コード内で、適切なプラグインの CSS <link>と JavaScript <script>を取り込むタグとタグを見つけます。以下のコードでは、IMA3 プラグインの例が強調表示されます。
  3. これらのパスをプレリリースバージョンに置き換えます。
  4. ページを更新すると、プレリリース版を使用します。
  5. テストが完了したら、記録されたパスを使用して JavaScript および CSS ファイルへのパスを再度編集し、元の設定を復元します。
