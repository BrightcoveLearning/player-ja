    プレイヤーイベント

    このトピックでは、Brightcove Player に関連するさまざまな種類のイベントについて説明します。

    概要

    Brightcove Playerには、ビデオの再生を制御するための多くのイベントがあります。このトピックでは、さまざまなタイプのイベントの概要を説明します。

    Brightcove Playerイベントの完全なリストは、プレーヤーメソッド/イベントAPI参照ドキュメント。このドキュメントは、プレーヤーを構成する実際のコードの一部であるイベントを反映しています。

    メディアイベント

    ザ・HTMLリビングスタンダードドキュメントは、Webアプリケーションに必要なHTMLとAPIの開発に関する情報の優れたリソースです。これは、Webの進化に関心を持つ人々の成長するコミュニティであるWebハイパーテキストアプリケーションテクノロジーワーキンググループ（WHATWG）によって最新の状態に保たれている「生きている」ドキュメントです。

    Brightcove PlayerはHTMLフレームワーク上で実行され、以下をトリガーしますメディアイベント。

    このドキュメントのイベントに関連するいくつかの用語は次のとおりです。

    ブロックされた

    再生が行われている場合、MediaControllerはブロックされていると見なされます一時停止。メディア要素は、そのコントローラーがブロックされたメディアコントローラー。

    MediaController

    A MediaController複数のメディア要素の再生を調整するオブジェクトです。

    メディア要素

    A メディア要素オーディオデータ、またはビデオとオーディオのデータをユーザーに提示します。

    奴隷

    デフォルトでは、各メディア要素にはMediaController。メディア要素がに関連付けられている場合MediaController、彼らは奴隷コントローラーに。コントローラは、それにスレーブされている各メディア要素の再生速度と音量を変更します。スレーブ要素の1つが予期せず停止した場合、コントローラーは他のスレーブ要素を停止します。

    バッファリングとQoSイベント

    バッファリングとサービス品質（QoS）に関連するイベントのサブセットは次のとおりです。

    イベント名 説明
    progress ビデオデータのバッファリング（ロード）
    waiting 要求されたビデオデータを一時的に待っています
    stalled バッファリングが停止しました
    error ビデオの読み込み中にエラーが発生しました
    playing 一時停止またはダウンロードの遅延後に再生が再開されました
    ratechange 再生速度が変更されました（手動または自動の可能性があります）
    IE11 / Edge Windows 10では、waitingバッファリングがある場合、イベントがディスパッチされることはありません。

    スタートアップイベント

    プレーヤーが初期化され、ビデオを再生する準備をするときに発生するイベントがいくつかあります。これらは、発送される順序でここにリストされています。

    • loadstart:プレーヤーが初期化されたときにディスパッチされ、再生する新しいソースを提供する場合に再初期化された場合

      以来loadstartイベントは、プレーヤーの準備が整う前に発生する可能性があります。プレーヤーのready()関数。以下のCodePenを参照してください。
    • loadedmetadata:プレーヤーが初期期間とディメンション情報を持っているとき、つまり最初のセグメントがダウンロードされたときにディスパッチされます。ライブビデオの場合loadedmetadataユーザーが[再生]をクリックするまで、イベントはディスパッチされません。これは、ユーザーが[再生]をクリックするまで、ライブビデオがセグメントのダウンロードを開始しないためです。
    • loadeddata:プレーヤーが現在の再生位置でデータをダウンロードしたときにディスパッチされます

    次のCodePenは、リッスンされてディスパッチされているスタートアップイベントを示しています。CodePenの上にマウスを置くと、クリックできることに注意してください。再実行右下にあるボタンをクリックすると、イベントが再度ディスパッチされます。コードを試してみたい場合は、 CODEPENで編集ヘッダーのリンクをクリックして、編集環境に移動します。あなたはクリックすることができますJSボタンをクリックして、イベントリスナーを追加するJavaScriptを表示します。

    ペンを見るスタートアップイベントのデモ Brightcove Learning Services（ @bcls） オンCodePen。

    注：このページを最初にロードすると、表示されない可能性があります。loadstartイベント。これは、CodePenがページに完全にレンダリングされる前にそのイベントがディスパッチされる可能性があるためです。クリック再実行ボタンをクリックすると、3つのイベントすべてがディスパッチされます。

    ready（）メソッド

    ザ・ready()メソッドは、イベントのように見えるという二重の性格を持っていますが、メソッドのように使用します。このメソッド/イベントは、プレーヤーがコマンドを受信する準備ができていることを意味します。

    ザ・ready()メソッドは、イベントリスナーとは異なります。readyイベントはすでに発生しています。ready()すぐにメソッド。Brightcove Playerを使用した開発の開始点を提供するために、次のように使用される方法がよく見られます。

    videojs.getPlayer('myPlayerID').ready(function(){
  var myPlayer = this;
});
    注：プレーヤーがビデオを再生するために、たとえばHTMLからFlashに再生テクノロジーを切り替える必要がある場合は、readyメソッド/イベントイベントハンドラーは、新しい再生技術が稼働すると再実行されます。

    ready（）とon（ 'loadedmetadata'、...）

    使用することに注意してくださいready()たとえば、プログラムでプラグインを追加するなど、プレーヤーと対話したい場合は正しく機能します。たとえば、ビデオをすぐに操作したい場合は、play()、上記のコードスニペットは常に正しく機能するとは限りません。より良いアプローチは、loadedmetadataたとえば、ビデオを操作するには：

    videojs.getPlayer('myPlayerID').on('loadedmetadata',function(){
  var myPlayer = this;
  myPlayer.play();
});

    要約すると、プレーヤーと対話するには、これを使用できます。

    videojs.getPlayer('myPlayerID').ready()

    プレーヤーでビデオをすぐに操作したい場合は、次を使用します。

    videojs.getPlayer('myPlayerID').on('loadedmetadata',function(){})
    注：あなたが待たなければならない理由loadedmetadataディスパッチされるイベントは、カタログが再生を試みる前にビデオを取得する時間を確保することです。

    フルスクリーンイベント

    A fullscreenchangeイベントは、フルスクリーンモードに切り替えられたとき、またはフルスクリーンモードから切り替えられたときにプレーヤーによって発行されます。プレーヤーがフルスクリーンに移行する場合でも、通常モードに戻る場合でも、同じイベントがブロードキャストされます。どちらが発生したかを知ることが重要な場合は、player.isFullscreen()プレーヤーが現在フルスクリーンモードであるかどうかを判断する方法。

    以下のCodepenは、これを行う方法を示しています。

    ペンを見るBrightcovePlayerフルスクリーンイベント Brightcove Learning Services（ @bcls） オンCodePen。

    広告のイベント

    プレーヤーに広告ライブラリと機能を含めるには、 IMA3プラグインまたはFreeWheelプラグイン。これは上に構築されていますvideojs広告フレームワーク（videojs-contrib-ads）。これらの広告プラグインはどちらも、次の表に示す広告イベントをディスパッチできます。それぞれに、実装に固有の特定のイベントもあります。

    イベント 次の場合に送出されます。
    広告リクエスト リクエストに応じて広告データ。
    広告ロード 広告リクエストの後に広告データが利用可能になる場合。
    ads-ad-開始 広告が再生され始めました。
    広告終了 広告の再生が終了しました。
    広告一時停止 広告が一時停止されています。
    広告プレイ 一時停止から広告が再開されます。
    広告第一四分位位 広告は総期間の 25% をプレイしています。
    広告の中点 広告は総期間の50％をプレイしました。
    広告三四分位数の 広告は総期間の 75% をプレイしています。
    広告クリック 視聴者が再生中の広告をクリックしました。
    ads-volumechange 再生中の広告の音量が変更されました。
    ads-pod-開始 リニア広告ポッド（一連の広告グループ）の最初の広告が開始されました。
    ads-pod-ENDD リニア広告ポッドの最後の広告（一連の広告グループ）が終了しました。
    ads-allpods完了 すべてのリニア広告の再生が終了しました。

    bc-catalog-errorイベント

    ready()スクリプトブロックの通常のセクションでエラーを処理すると、問題が発生する可能性があります。たとえば、bc-catalog-errorプレイヤーが準備が整う前にイベントが送出され、ready()セクション内のエラーをリッスンすると、エラーを処理できなくなります。この問題は、地理フィルタリングを使用する場合、ビデオが公開されていない、ビデオがスケジュール範囲外の場合、または別のアカウントで発生する可能性があります。完全な議論と取り扱い例についてはbc-catalog-errorイベント、を参照してくださいプレイヤーカタログ資料。

    進捗状況とtimeupdate

    の違いについて混乱が生じる可能性がありますprogressそしてtimeupdateイベント。Brightcove Playerは、HTML5ビデオの上に構築されており、その標準では、progressブラウザがデータをダウンロードしているときにイベントがディスパッチされます。ザ・timeupdate現在の再生位置が変更されると、イベントがディスパッチされます。

    これは、そのAPIのように、BrightcoveのSmartPlayerのユーザーを混乱させる可能性があります。progressイベントは何をしますかtimeupdate Brightcove PlayerAPIで行います。

    timeupdateリスナーの削除に関する注意

    場合によっては、に追加したイベントリスナーを削除したいことがあります。timeupdateイベント。ユースケースはに示されていますBrightcoveプレーヤーのサンプル：プレビュー後に再生するための登録資料。この状況では、プレーヤーを特定の時間範囲で一時停止する必要があり、timeupdate時間チェックには、匿名のイベントハンドラー関数定義を使用します。一時停止するのは1回だけなので、イベントリスナーを削除する必要があります。あなたは単にしたくない：

    myPlayer.off('timeupdate');

    もちろん、これによりすべてのイベントリスナーが削除されますtimeupdate、および他の問題の中でも、これを行うと、タイムスクラバーが進むのを防ぐことができます。実行する必要があるのは、通常の関数定義構文（関数宣言）を使用して関数を作成してから、1つのイベントリスナーだけを削除することです。省略されたコードはここに示されています：

     // Add the event handler
myPlayer.on('timeupdate', timeupdateHandler);

// Handle the event then remove JUST this event listener on timeupdate
function timeupdateHandler(evt) {
  ...
  myPlayer.off('timeupdate', timeupdateHandler);
}

    分析イベント

    ビーコンがAnalyticsデータコレクターに送信されるたびにイベントがトリガーされます。ビーコンまたは特定のビーコンをリッスンできます。

    すべてのビーコンを聞く

    をリッスンすることで送信されたすべての分析ビーコンを追跡できますanalytics_requestイベント：

    player.on('analytics_request', function(e) {
  videojs.log('sent a(n) ' + e.params.event + ' beacon!', e.params);
});

    特定のビーコンを聞く

    をリッスンすることで送信された特定の分析ビーコンを追跡できますanalytics_request_{beacon_name}イベント。

    {beacon_name} is the name of any of the player events that are sent to the data collector.これらのイベントの完全なリストについては、データ収集APIリファレンス。

    例

    player.on('analytics_request_video_impression', function(e) {
  videojs.log('sent an impression beacon!', e.params);
});

    player.on(event, function () {})
.on(event, function () {})
.on(event, function () {})
.on(event, function () {});
