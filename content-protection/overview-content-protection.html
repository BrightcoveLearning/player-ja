DRM

DRM（デジタル著作権管理）は、ビデオデータを暗号化し、ライセンスポリシーに基づいてロックを解除することにより、ビデオコンテンツを保護します。動画コンテンツが Brightcove Player にロードされると、プレーヤーはライセンスサーバーにコールバックし、動画を再生する許可を取得します。このトランザクションは閲覧者には透過的です。概要を参照してください。詳細については、Video Cloud のデジタル著作権管理 (DRM) ドキュメントを参照してください。

ジオフィルタリング

地理フィルタリングは、コンテンツ保護に使用できます。Video Cloud ビデオをジオフィルタリングする方法については、「ビデオ可用性の設定」ドキュメントを参照してください。地理フィルタリングエラーに対処する方法については、「ジオフィルタリングメッセージング」ドキュメントを参照してください。Brightcove Player。

TTL（トークン認証）

プレミアムコンテンツまたは機密コンテンツを配信する場合は、そのコンテンツを安全に保つためのあらゆる方法を探すことになります。適切な承認なしに、エンドユーザーがビデオ URL をコピーしたり、コンテンツの再生を継続したりすることは望ましくありません。ブライトコーブは、HLS、MP4、および DASH URL の VOD 署名をサポートしています。署名は、Akamai、Fastly、Cloudfront、および JOCDN を使用したブライトコーブハウスと BYO CDN 設定の両方でサポートされています。

デフォルトでは、TTLトークンの寿命は非常に短くなります。必要に応じて（MRSS フィードやその他のユースケースの場合）、ブライトコーブサポートに連絡してトークンの有効期間の値を高く設定することで、寿命を延ばすことができます。指定できる範囲は 1 時間から 1 年です。ただし、これはアカウント設定であることを覚えておいてください。他の動画のセキュリティを危険にさらす可能性があるため、この値を高く設定しないでください。

ドメインの制限

ドメイン制限をプレイヤーに設定して、プレイヤーが使用できるドメインを制限することができます。プレーヤーの公開コードをコピーして別のサイトで使用すると、ドメイン制限によってプレーヤーが動画を読み込めなくなります。ホワイトリストに登録されているドメインで実際に再生を実行するには、パブリッシャーがページ内埋め込みコードを使用する必要があります。パブリッシャーは、ホワイトリストに登録されたドメイン（または他のドメイン）で iframe 埋め込みコードを使用できず、再生が正常に機能することを期待しています。

プレーヤーをドメイン制限する方法については、「プレーヤーコンテンツ制限の設定」ドキュメントと、ドメイン制限への対応方法については、「ドメイン制限メッセージング」ドキュメントを参照してください。Brightcove Playerのエラー。

IP の制限

IP 制限を使用すると、米国外のホワイトリストに登録された IP アドレスが Video Cloud コンテンツにアクセスできるようにプレーヤーを設定できます。サポートについては、カスタマーサポートにお問い合わせください。IP 制限のある北米以外で使用しているプレーヤーについては、「IP アドレスを使用したビデオ再生の制限」の情報を必ずチェックして、発生する可能性がある問題を回避してください。
