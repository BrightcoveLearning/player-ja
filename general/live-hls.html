ソースの割り当て

実装は通常の手順に従います。次のサンプルに示すように、プレーヤーを作成し、ライブ HLS URL をソースとして指定します。注:ソースにはがあり.m3u8、タイプはですapplication/x-mpegURL。

curl \
  --header "Content-Type: application/json" \
  --user $EMAIL \
  --request POST \
  --data '{
      "name": "MySamplePlayer",
      "configuration": {
        "media": {
          "sources": [{
            "src":"http://bcoveliveios-i.akamaihd.net/hls/live/215102/master_english/398/master.m3u8?playerId=1111&lineupId=&affiliateId=&pubId=3333&videoId=2222",
            "type":"application/x-mpegURL"
          }]
        }
      }
    }' \
    https://players.api.brightcove.com/v2/accounts/$ACCOUNT_ID/players

既知の問題

ライブ HLS と新しいプレーヤーに関する既知の問題は次のとおりです。

  • iOSはネイティブプレーヤーを使用してライブHLSをストリーミングします。
  • iOSでは、ライブストリームでプレイをヒットすると、フルスクリーンになります。新しいプレイヤーはそれを制御できない。
  • 現在、新しいプレーヤーは HLS ストリームがライブかどうかを検出しないため、Android ではライブ UI が適用されません。
