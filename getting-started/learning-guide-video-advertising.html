広告タイプ

ビデオ広告には主に3つのタイプがあります。

  • リニア広告 :リニア広告は動画の再生中に発生し、プレーヤー自体に表示されます。広告は、ユーザーによる動画コンテンツの消費前（プリロール）、ミッドロール（ミッドロール）、後（ポストロール）のいずれかにビデオの再生を中断します。以下のサンプルビデオは、IMA プラグインの使用方法を示しています。ビデオを再生してプリロール、5秒でスキップ可能なミッドロール、最後にポストロールを確認します。
  • ノンリニア広告 :ノンリニア広告は再生中に掲載されるため、ユーザーはコンテンツを見ながら広告を見ることができます。ノンリニア広告には、テキスト、グラフィック広告、動画オーバーレイもあります。ノンリニア広告の例としては、再生中の動画の上に表示されるが、動画が中断されないテキストオーバーレイがあります。次の例は、ユーザーがクリックして別のページに移動できるオーバーレイのスクリーンショットを示しています。オーバーレイは指定された時間表示されますが、ビデオの再生は中断されません。
  • コンパニオン広告：コンパニオン広告は、動画プレーヤーの周囲に表示されるテキスト広告またはディスプレイ広告で、通常は動画コンテンツに関連付けられます。コンパニオン広告の例としては、動画コンテンツに合わせて調整されたバナーがあります。次の例はスクリーンショットコンパニオン広告のスロットの、赤い破線でマークされ、バナーで埋められています。

広告スタンダード

Interactive Advertising Bureau（IAB）は、650社以上の大手メディアおよびテクノロジー企業による広告事業組織で、業界標準を開発し、調査を行い、オンライン広告業界に対する法的支援を行っています。

IAB ビデオスイートのビデオは、Whys、hows、hows、またはそれらの基準を説明する優れた仕事をします。

動画広告の主な基準は次のとおりです。

  • 動画複数広告プレイリスト (VMAP) :動画のプレロール、ミッドロール、ポストロール、ノンリニア広告、およびそれらを表示するタイミングについて説明します。
    • IAB の基準から:「IAB 動画複数広告プレイリスト（VMAP）の仕様は、動画コンテンツ所有者が、動画プレーヤーやコンテンツ配信アウトレットを制御していない場合に、広告インベントリの挿入構造を説明するために使用できる XML テンプレートです。「
  • 動画広告配信テンプレート (VAST) :VMAPはプリロールがあると言い、VAST はそのプリロールを記述している。
    • IAB の基準から:「動画広告配信テンプレート（VAST）には、インストリーム動画用の標準の XML ベースの広告レスポンス、開発者向けの XML スキーマ定義（「XSD」）が含まれています。これは、オンラインデジタルビデオ広告事業における現在の慣行のほとんどに対応することを意図しています。「
  • ビデオプレーヤー広告配信インターフェイスの定義 (Vペイド) :Vペイドは、インタラクティブ広告ではVAST のようなものです。
    • VPAID:IAB の標準から」「動画プレーヤー広告配信インターフェイス定義（Vペイド）」は、動画プレーヤーと広告ユニット間の共通のインターフェースを確立し、インタラクティブなインストリーム広告体験を可能にします。「
    • 簡単なVペイドの例としては、短いイントロビデオが表示され、[ 詳細情報] テキストの上にマウスを置くと、長いビデオが再生されます。
    • Vペイド形式は、広告代理店や広告主が、視認性スクリプトを実行するために一般的に使用されます。

規格の完全なドキュメントについては、 IAB の Web サイトを参照してください。

広告ポッド

VAST 3.x の新機能には、広告ポッドの概念があります。広告ポッドは、一連のシーケンシャルリニア広告の配信です。以下の図は、通常の動画コンテンツよりも前に連続して再生される 3 つの広告を含む広告ポッドを示しています。広告ポッドは、コンテンツビデオの再生前、侵入中、または再生後に再生でき、複数の広告スポットがあるテレビコマーシャルのように機能します。

VMAPを使用して、広告ポッドのブレークを指定できます。

広告イベントと広告オブジェクトには、広告ポッドの再生に関するイベントが表示されます。

グーグルのIMA SDK

Google は動画広告の主要プレーヤーであり、パブリッシャーが動画やゲームにリニア広告、ノンリニア広告、コンパニオン広告を表示できる Google インタラクティブメディア広告（IMA）SDK を定義しています。SDK を使用すると、開発者は DoubleClick for パブリッシャー (DFP)、Google AdSense ネットワーク、または VAST 準拠の広告サーバーに動画広告リクエストを行うことができます。Brightcove Player用の IMA3 プラグインは、Brightcove Playerと Google のインタラクティブメディア広告 (IMA) バージョン 3 を HTML5 および Flash 用に統合しています。これにより、Brightcove Player の広告をリクエストして追跡することができます。

理解するための重要なIMA用語は次のとおりです。

  • 広告ルール :動画コンテンツに広告を表示するタイミングと場所、方法を定義します。広告ルールを作成して、動画内またはセッション中（複数の動画ストリームにわたって）広告をいつ再生するか、どのくらいの期間、およびトリガーの結果として表示するかを決定できます。広告ルールは、VMAP ドキュメントとほぼ同じです。
  • 広告タグ :広告サーバーから広告を取得するための指示と、広告をどうするかを、ブラウザに指示します。指示は広告リクエストを生成し、広告サーバーからの適切な応答があります。広告タグは、VAST ドキュメントとほぼ同じです。

IMA の詳細については、Google の開発者ヘルプセクションの「インタラクティブメディア広告」を参照してください。

動画広告の実体

動画広告に関与するエンティティには多数のグループが存在しますが、非常に簡単な例を次に示します。

  • 出版者-動画コンテンツを表示して、広告を使用して収益を上げたいと考えています。
  • 広告主-顧客/潜在的な顧客に見てもらいたい広告を用意します。
  • 広告ネットワーク-' ' パブリッシャーの広告スペースを広告主の広告と一致させます。
  • ビデオテクノロジープロバイダー-パブリッシャーと広告ネットワークにエンコーディングとプレーヤーテクノロジーを提供します。ブライトコーブは、動画テクノロジープロバイダーグループに属します。

フラッシュ対HTML

現時点では、Flashは最も頻繁に使用される広告フォーマットとしてHTML5に道を譲っています。このため、adTechOrderBrightcove Playerのオプション用の IMA3 プラグインを html5 に設定してから、フラッシュすることをお勧めします。。詳細については、「IMA でフラッシュが無効になった」を参照してください。

Brightcove Playerでの広告ワークフロー

広告を Brightcove Player と連携させる場合の基本的な広告ワークフローは次のとおりです。

  1. 広告プロバイダにお問い合わせください。 広告関連の Brightcove パートナーはすべて、パートナーページからこの検索で見つけることができます。ブライトコーブを使用する広告プロバイダーは、広告テクノロジーをサポートするためにプラグインを作成した Google の IMA3 テクノロジーと FreeWheel です。他の多くの広告プロバイダーが Brightcove Player 用の独自のプラグインを作成しています。次の表に、さまざまな広告プロバイダーと広告関連テクノロジーの詳細と、それらが Brightcove Player とどのように関連しているかを示します。プロバイダ/テクノロジーIMA3プラグイン。
    プロバイダ/テクノロジー コメント
    DoubleClick (IMA3) IMA3 プラグインでサポートされています。
    フリーホイール FreeWheel プラグインでサポートされています
    SpotX プラグインは SpotX から直接入手できます。SpotX の Brightcove プラグイン統合ドキュメントページを参照してください。
    サードパーティのVMAP（IMA3プラグイン経由） VMAP は、プリロール、ミッドロール、ポストロール、コンパニオンをサポートしています。
    サードパーティの VAST3 (IMA3 プラグイン経由) APIは、VAST URL経由で広告を注入することができます。
    Vペイド 2 広告 (IMA3 プラグイン経由) Brightcove Playerは Javascript Vペイド 2.0 広告ユニットをサポートしています。つまり、Flash を使用せずにすべてのデバイスに展開できます。

    サーバー側広告挿入 (SSAI) を使用します。 ブライトコーブの SSAI プラグインを使用して、動画ストリームに広告をステッチすることもできます。この機能は、動的配信で有効になります。詳細については、次のドキュメントを参照してください。

    • ビデオクラウド SSAI の概要
    • SSAI プラグインを使用した広告
  2. 広告タグを確認します。広告プロバイダーは、Brightcove Player に提供する URL を提供します。プレーヤーの統合を開始する前に、広告タグが機能していることを確認することをお勧めします。広告プロバイダーが標準の VAST 広告または VMAP 広告の回答を使用している場合は、Google の動画スイートインスペクタを使用して、広告タグが機能しているかどうかをすばやくテストできます。他のベンダーの中には、同様のテクノロジを提供しているものもあります。URL の検証は、初期設定時のベストプラクティスであるだけでなく、後で問題が発生した場合は、広告の再生のトラブルシューティングの最初の手順として、URL を再テストします。
  3. ベンダーの情報を使用して Brightcove Player を設定します。 広告プロバイダーの URL の有効性を確認したら、適切なプラグインを使用して Brightcove Player を広告サーバーに接続します。
  4. 設定をテストします。 すべてのプラットフォーム、OS、ブラウザーで広告が再生されるようにする必要があります。たとえば、広告プロバイダーが Flash 広告のみを提供している場合、iOS デバイスで問題が発生する可能性があります。また、デフォルトのデスクトップブラウザの設定により、広告の動作が変更されます。
