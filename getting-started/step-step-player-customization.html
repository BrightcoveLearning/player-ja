    ステップバイステップ:プレーヤーのカスタマイズ

    このトピックでは、に従って Brightcove Player の外観をカスタマイズするための実践的な手順について説明します。Brightcove Player の外観をカスタマイズするには、カスケードスタイルシート（CSS）を使用する必要があります。このドキュメントでは、最初にビデオをオーバーレイする再生ボタンの外観と場所を変更する手順について説明します。

    概要

    この文書には、ページ内埋め込みで CSS を使用し、プレーヤーの iframe 実装はかなり異なるため、いくつかのセクションがあります。

    このドキュメントのコンテンツは Chrome の開発ツールを使用していますが、最新のブラウザーはすべて同等の機能を備えています。さまざまなブラウザーでの開発ツールの使い方の簡単な概要については、「デバッグの基本」ドキュメントを参照してください。

    注：いくつかの基本的なプレーヤーのスタイリングは、Video Cloud Studioの内部で提供されています。詳細は「プレーヤーのスタイルを設定する」を参照してください。

    プレイヤーセレクターを決定する

    CSS セレクタを使用すると、ページ上の HTML 要素を選択して操作できます。CSSセレクタは、そのID、クラス、タイプ、属性などに基づいて HTML 要素を「検索」（または選択）するために使用されます。HTMLページ上の要素の外観を変更しようとするときに最初に行うべきことは、要素のアドレス指定に使用するセレクタを決定することです。次の手順は、プレーヤーのセレクターを検索し、プレーヤーのサイズを変更する方法を示しています。

    1. Video Cloud Studio で Media モジュールを開き、公開するビデオを選択します。
    2. ビデオを再生するプレーヤーを選択します。
    3. をセットするサイジング、アスペクト比とプレーヤーのサイズ。
    4. をコピーします高度なコードの公開。
    5. HTML エディタを使用して、新しい HTML ページを作成します。
    6. 発行コードを HTML ページの本文に貼り付けます。
    7. 新しく作成したページを参照します。
    8. ブラウザの開発ツールを開き、HTMLにドリルダウンして、<video-js>鬼ごっこ。が表示されます<video-js>タグには、という名前のクラスがありますvideo-js。ザ・<video-js>タグには常に次のクラスがありますvideo-js。あなたは利用しますvideo-jsプレーヤーをスタイリングするときに頻繁にクラスを作成します。
    9. 下のスクリーンショットをチェックして、赤でハイライトされたクラスを確認してください。緑で強調表示されているのは<video-js>タグ、もちろん、BrightcovePlayerです。

    再生ボタンセレクタ

    別のアプローチは、再生ボタンのセレクタを見つけるために使用されます。次の手順は、再生ボタンの正しいセレクタを検索し、その外観を変更する方法を示しています。

    1. HTML ページを参照します。
    2. プレーヤーをオーバーレイする再生ボタンを右クリックし、[ 要素を検査 ] を選択します。
    3. 注意してください<div>強調表示されているタグは再生ボタンを表し、クラスはvjs-big-play-button。
    4. 再生ボタンの色と不透明度を変更するCSSを含めるようにスタイルを変更します。 
      <style>
  .vjs-big-play-button {
    background-color: red;
    opacity: .3;
  }
</style>
    5. ページを参照すると、すべてのスタイルが使用されているとは限りません。ボタンは赤ではありません。
    6. 開発者ツールの右側を見ると、スタイルの概要が表示されます。下を見ると、背景を赤に変更すると上書きされ、背景が交差していることがわかります。
      注：Safari では、スタイルがオーバーライドされているにもかかわらず、そのスタイルはクロスアウトされない場合があります。

      何が起こったかは、背景色が多数の変更が行われたことです。設定したスタイルは、CSS の「カスケード」部分でオーバーライドされています。これは、特異性と呼ばれるCSSの機能を利用することで変更できます。特異性は、ブラウザによって適用される CSS ルールを決定します。また、カスケードスタイルシート (CSS) ルールの表示方法に影響する重み付けの一種です。特異性を使用して、あなたのスタイルを他のものよりも重要にし、強制的に使用することができます。にCSSの特異性についての簡単な紹介がありますプレーヤーの外観をカスタマイズする doc、および任意の検索での多数の参照。

    7. この場合、video-jsクラスはあなたのスタイルを「ブースト」して優先します。スタイルを変更して、新しい組み合わせセレクタを使用します。最後のクラスの前に1つのスペースがあることに注意してください（理由の詳細は、特異性のルールに記載されています）。 
      <style>
  .video-js .vjs-big-play-button {
    background-color: #ff0000;
    opacity: .3;
  }
</style>
    8. ページを参照して、すべてのスタイルが適用されているかどうかを確認します。
    9. もう一度開発者ツールの右側を見ると、スタイルの概要が表示されます。これで、背景色の変更が上書きされないことがわかります。

    高度なコード実装

    上記のセレクターとスタイルを使用する場合、結果は、次に示すように、プレーヤーの高度なコード実装で使用されました。

    <!doctype html>
<html>

<head>
  <meta charset="UTF-8">
  <title>Untitled Document</title>
  <style>
    .video-js .vjs-big-play-button {
      background-color: red;
      opacity: .3;
    }
  </style>
</head>

<body>

  <video-js data-account="1507807800001"
    data-player="default"
    data-embed="default"
    controls=""
    data-video-id="6065136830001"
    data-playlist-id=""
    data-application-id=""
    width="640" height="360">

  </video-js><script src="https://players.brightcove.net/1507807800001/default_default/index.min.js"></script>

</body>

</html>

    これは、HTMLページでCSSを使用する非常に一般的な方法です。あなたが見る<style>ブロックはページ上にあります<video-js>鬼ごっこ。

    注：CSSの経験によっては、上記のコードで割り当てられなかった理由がわからない場合があります。idに属性<video-js>鬼ごっこ。その後、#myPlayer .vjs-big-play-buttonのようなセレクタを使うことができます。実際、あなたは持っていることができ、それはうまくいくだろう。問題は、iframe Player の実装を使用するときに id セレクターを使用して作成された CSS が機能しないため、既存のクラス名のみが使用されたことです。id属性を使用しないことで、スタイルはiframeプレーヤーの実装に直接適用されます。

    標準コードの実装

    標準またはiframeのコード実装は、当然のことながら、Brightcove Playerをiframeに配置し、次にHTMLページに配置します。実際のiframe自体のスタイルを設定したり、例えばiframeを含むページのスタイルを使用してサイズを変更することはできますが、iframeのページ上のスタイルはiframe自体にカスケードされません。iframeは文字通り別のHTMLページを取り込むので、iframeの内容の必要なスタイルは、iframe内のHTMLページに関連付ける必要があります。

    Brightcove ブプレーヤーの iframe 実装を使用する場合、CSS を適用する手順は次のとおりです。

    • インターネットにアクセス可能なファイルにスタイルを配置する
    • スタイルシートをプレーヤーにリンクする (Studio またはプレーヤー管理 API を使用して実行)

    注：以下の手順は、前の手順で完了し、上記のコードスニペットに示された HTML から作業することを前提としています。それを実例として残したい場合は、コードを別のファイルにコピーし、新しいファイルで次の手順を開始します。

    1. Studio で、ビデオを公開し、プレーヤーの iframe 実装用のコードをコピーします。
    2. HTML ページで、ページ内埋め込みコードを iframe コードに置き換えます。
    3. HTML ページが次のようになっていることを確認してください。 
      <!doctype html>
<html>

<head>
  <meta charset="UTF-8">
  <title>Untitled Document</title>
  <style>
    .video-js .vjs-big-play-button {
      background-color: red;
      opacity: .3;
    }
  </style>
</head>

<body>

<iframe src="https://players.brightcove.net/1507807800001/default_default/index.html?videoId=6065136830001"
  allowfullscreen=""
  allow="encrypted-media"
  width="640" height="360"></iframe>

</body>

</html>
    4. ページを参照すると、ボタンのスタイルが使用されていないことがわかります。
    5. customize-qs.css という名前のファイルを作成し、インターネットにアクセスできる場所に保存します。
    6. ボタンのスタイルをカットし、ファイルに貼り付けて保存します。ボタンのスタイルのみをカットします。含めることはできません<style>タグ。このファイルには、次の内容のみが含まれている必要があります。 
      .video-js .vjs-big-play-button {
  background-color: red;
  opacity: .3;
}
    7. HTML ページが次のようになっていることを確認してください。 
      <!doctype html>
<html>

<head>
  <meta charset="UTF-8">
  <title>Untitled Document</title>

<body>

<iframe src="https://players.brightcove.net/1507807800001/default_default/index.html?videoId=6065136830001"
  allowfullscreen=""
  allow="encrypted-media"
  width="640" height="360"></iframe>

</body>

</html>
    8. Studioで、 Players モジュールに移動し、このクイックスタートで使用しているプレーヤーを編集します。
    9. 左側のナビゲーションメニューで [ プラグイン ] をクリックします。
    10. クリックスタイルシート、 そうしてスタイルシートを追加するボタンをクリックし、パスを入力します。
    11. [保存]をクリックして、変更を保存します。ページの右側にあるプレビュープレーヤーが更新され、ボタンに適用されているCSSスタイルが表示されます。
    12. プレーヤーを公開するには、[ パブリッシュと埋め込み] [> 変更を公開 ] をクリックします。
    13. 開くダイアログを閉じるには、[ 閉じる ] をクリックします。
    14. ブラウザに戻り、ページを更新します。パブリッシュされた変更が有効になると、iframe プレーヤーに適用されたスタイルが表示されます。パブリッシュされた変更が表示されるまでに、数分かかる場合があります。

    スタイルを探る

    このセクションでは、CodePenを使用して、スタイルを探索し、上に示したものよりも大きな再生ボタンをさらにカスタマイズできるようにします。あなたが好きな外観を取得したら、あなたは単に生成されたCSSをコピーし、上記のようにいずれかのプレーヤーの実装でそれを使用することができます。

    混乱しているように見える1つの問題は、ボタンの実際の矢印が画像ではなく、フォントファミリの一部であることです。そのため、font-sizeを使用してサイズを制御し、ems単位として使用します。

    CodePen の Pen 5.x ビッグプレイボタンのカスタマイズマット・ボレス ( @mboles ) を参照してください。
