    FreeWheel プラグインでの広告

    このトピックでは、FreeWheelプラグインを使用して、ビデオ再生で広告を有効にする方法と、Studioを使用してカスタムコーディングを介して実装する方法を学習します。

    概要

    このプラグインは、Brightcove ブプレーヤーで Flash または HTML5 の FreeWheel 広告テクノロジーを有効にします。FreeWheelプラグインのデフォルトのレンダリングオプションはに設定されていますHtml5（詳細については、adTechOrder以下のオプション）。

    プラグインのバージョンを使用する

    FreeWheelプラグインを使用する場合は、バージョン管理されたURLを使用する必要があります。

    バージョン タイプ バージョン管理されたURL
    最新のリリース JS https://players.brightcove.net/videojs-freewheel/3/videojs-freewheel.min.js
    CSS https://players.brightcove.net/videojs-freewheel/3/videojs-freewheel.min.css
    SWF * https://players.brightcove.net/videojs-freewheel/3/videojs.freewheel.swf

    参考までに、プラグインと互換性のあるSWFのバージョンはプラグインバージョンリファレンス資料。

    広告サーバーをテストする

    最初に行うべきことは、使用する予定の広告タグの有効性を検証することです。URL をコピーしたことを確認し、次のページを参照します。広告プレビューア（このリンクをクリックすると、ページが新しいウィンドウまたはタブで開きます）。

    指定されたフォーム入力フィールドに広告タグのURLを貼り付けます。クリックOPEN IN AD PREVIEWER。タイトルのポップアップOpen In Ad Previewerが表示されたら、をクリックしますOPEN広告をテストするためのボタン。クリックPLAYまた、正しく機能している場合、広告には動画がちりばめられて表示されます。このテスト環境で広告タグが機能しない場合は、Brightcove Player では機能しません。

    正しいSDKURLを使用する

    SDKバージョン6.24より前では、FreeWheelは最新バージョンのJS AdMangerSDKへの「自動アップグレード」URLを提供していました。このURLは、常に最新バージョンのSDKを指していました。これはもはやそうではありません。

    次の2つのセクションでは、FreeWheelを実装する代わりの方法を提供します。両方のセクションで、sdkurl。過去には、単に使用することができましたhttp://adm.fwmrm.net/p/[]/AdManager.jsオプションの値については、FreeWheelSDKの最新バージョンを指していることを知ってください。次に、次のような特定のバージョンを指すURLを入力する必要があります。https://mssl.fwmrm.net/libs/adm/6.24.0/AdManager.js。加えて、初期設定後、最新バージョンを使用したい場合は、FreeWheelによってSDKの新しいバージョンがリリースされたときにその値を更新する必要もあります。この変更を行った後、Brightcoveプレーヤーを再公開する必要があることに注意してください。

    URLを更新せずに、自動アップグレードURLを引き続き使用すると、バージョン6.23を永久に指すことになります。

    Brightcove Player インテグレーションでは、独自の SDK バージョンを管理することができます。これには、Html5.sdkurlオプションを FreeWheel プラグインに渡すことが含まれます。

    player.FreeWheelPlugin({
  Html5: {
    sdkurl: 'preferred-sdk-url'
  }
});

    プレーヤーモジュールを使用して実装する

    Playersモジュールを使用してFreeWheelプラグインを実装するには、次の手順に従います。

    1. PLAYERS モジュールを開き、新しいプレーヤーを作成するか、プラグインを追加するプレーヤーを見つけます。
    2. プレーヤーのリンクをクリックして、プレイヤーのプロパティを開きます。
    3. 左側のナビゲーションメニューで [ プラグイン ] をクリックします。
    4. 次に、3 つのオプションから [ プラグイン ] をクリックします。
    5. [ プラグインの追加 ] から [ カスタムプラグイン ] を選択します。
    6. [ プラグイン名 ] に、と入力しますFreeWheelPlugin。
    7. JavaScript の URL には、次のように入力します。 
      https://players.brightcove.net/videojs-freewheel/3/videojs-freewheel.min.js
    8. CSS の URL には、次のように入力します。 
      https://players.brightcove.net/videojs-freewheel/3/videojs-freewheel.min.css
    9. [ オプション (JSON) ] テキストボックスに構成オプションを入力します。フォームで提供されるオプションは、再生テクノロジーを構成します。FlashそしてHtml5構成セクションはFreeWheelによって提供されます。セキュリティ上の理由から、少し変更された、完全な設定を以下に示します。スタブの例を次に示します。 
      {
  "adTechOrder": ["html5"],
  "Flash": {
    "sdkurl": "https://mssl.fwmrm.net/libs/adm/{LATEST_VERSION}/...",
    "serverUrl": "http://cue.v.fwmrm.net/..."
    "profile": "3aaa_profile",
    ...
  },
  "Html5": {
    "sdkurl": "https://mssl.fwmrm.net/libs/adm/{LATEST_VERSION}/...",
    "serverUrl": "http://cue.v.fwmrm.net/..."
    "profile": "3aaa_profile",
    ...
  }
}
    10. [保存]をクリックします。
    11. プレーヤーを公開するには、[ パブリッシュと埋め込み] [> 変更を公開 ] をクリックします。
    12. 開くダイアログを閉じるには、[ 閉じる ] をクリックします。

    コードを使用して実装する

    プラグインを実装するには、プラグインコードの場所、必要に応じてスタイルシート、プラグイン名、プラグイン設定オプションを知る必要があります。プラグインコードとスタイルシートの場所は次のとおりです。

    https://players.brightcove.net/videojs-freewheel/3/videojs-freewheel.min.js
https://players.brightcove.net/videojs-freewheel/3/videojs-freewheel.min.css

    {

  Flash: {
    sdkurl: "https://mssl.fwmrm.net/libs/adm/{LATEST_VERSION}/...",
    serverUrl: "http://cue.v.fwmrm.net/..."
    profile: "3aaa_profile",
    ...
  },
  Html5: {
    sdkurl: "https://mssl.fwmrm.net/libs/adm/{LATEST_VERSION}/...",
    serverUrl: "http://cue.v.fwmrm.net/..."
    profile: "3aaa_profile",
    ...
}

    レンダリングオプション

    次の表は、FreeWheelプラグインがBrightcove Player（列1）での技術選択に基づいて広告（列3）をレンダリングする技術とadTechOrderプラグインで設定します（列2）。

    Brightcove ブプレーヤー
    選択された技術    		 FreeWheelPlugin adTechOrder FreeWheelプラグイン
    選択された広告技術
    html5 ["flash","html5"] flash
    html5 ["html5","flash"] html5
    html5 ["html5"] html5
    html5 ["flash"] flash
    flash ["flash","html5"] flash
    flash ["html5","flash"] flash
    flash ["html5"] ありえない
    flash ["flash"] flash
    注：ミッドロールを配信するときに最高のユーザーエクスペリエンスを得るには、HLSなどのセグメント化された配信形式を使用し、バイト範囲リクエストをサポートするCDNからコンテンツを配信する必要があります。

    設定

    FreeWheelプラグインの構成には、次の設定が使用されます。

    • adTechOrder:
      • タイプ:Array Default Value: ["html5"]
      • 配列には、優先順位の高い順に、使用を試みる広告統合テクノロジーが含まれています。
    • Flash:
      • タイプ: Object
      • 技術固有の FreeWheel 設定のオブジェクト (FreeWheel によって提供)。オブジェクトには、次のオプションがあります。
        これらのオプションは、動的に設定（ここで表示）することも、初期設定の一部として設定することもできます（ここを参照）。
      • swfurl:
        • タイプ:string Default Value: https://players.brightcove.net/videojs-freewheel/3/videojs.freewheel.swf (compatible with FW Flash SDK v6.4.6)
        • オプションの値は、FreeWheelFlashプラグインのバージョンに基づいて使用できます。これらの値は、Using versions of the pluginこのドキュメントの上部にあるセクション。
        • Flashオブジェクトの子プロパティである必要があります。Flash 広告技術が使用されている場合、これはオブジェクト要素に直接適用されます。
      • autoplay:

    イベント

    プラグインは、ロード、初期化、および再生中にいくつかのカスタムイベントタイプを放出します。他のイベントと同じように、広告フレームワークのイベントをリッスンできます。

    player.on('ads-ad-started', function(event) {
  console.log('event', event);
});
    イベント 次の場合に送出されます。
    広告リクエスト リクエストに応じて広告データ。
    広告ロード 広告リクエストの後に広告データが利用可能になる場合。
    ads-ad-開始 広告が再生され始めました。
    広告終了 広告の再生が終了しました。
    広告一時停止 広告が一時停止されています。
    広告プレイ 一時停止から広告が再開されます。
    広告第一四分位位 広告は総期間の 25% をプレイしています。
    広告の中点 広告は総期間の50％をプレイしました。
    広告三四分位数の 広告は総期間の 75% をプレイしています。
    広告クリック 視聴者が再生中の広告をクリックしました。
    ads-volumechange 再生中の広告の音量が変更されました。
    ads-pod-開始 リニア広告ポッド（一連の広告グループ）の最初の広告が開始されました。
    ads-pod-ENDD リニア広告ポッドの最後の広告（一連の広告グループ）が終了しました。
    ads-allpods完了 すべてのリニア広告の再生が終了しました。
    fw-before-ad-request このイベントはプレーヤーオブジェクトで公開され、広告リクエストを送信する前にトリガーされます。通常、プレイリストのコンテキストで使用され、次の方法でFreeWheel構成設定を更新します。player.FreeWheelPlugin.settings.Html5またはplayer.FreeWheelPlugin.settings.Flash

    動的サーバーURL割り当て

    あなたは使用することができますfw-before-ad-requestサーバーのURLを動的に割り当てるイベント。を使用しますon()広告リクエストをリッスンするメソッド。次に、目的のサーバーURLを割り当てます。もちろん、コード内のプレースホルダーに必要なサーバーURLを指定する必要があります。

    player.on('fw-before-ad-request', function () {
  player.FreeWheelPlugin.settings.Html5.serverUrl = '[your server url]'
  player.FreeWheelPlugin.settings.Flash.serverUrl = '[your server url]'
})

    以前にサーバーのURLを構成したことがある場合、表示されるコードは以前の構成を上書きします。

    既知の問題

    FreeWheelで同じプレーヤーを動的に再利用する

    ビデオを動的にロードして同じプレーヤーを再利用する場合、FreeWheel構成設定には、2つの例外を除いて、FreeWheelプラグインがインスタンス化されたときに設定されたものと同じ値が含まれます。最初の例外は、広告マクロを使用している場合です。2番目の例外は、を使用して設定を更新できることです。player.FreeWheelPlugin.settings.Html5またはplayer.FreeWheelPlugin.settings.Flashにfw-before-ad-requestイベント。

    広告再生中のプレーヤーのサイズ変更

    広告または動画の再生中にプレーヤーのサイズが変更された場合、プレーヤーのサイズ変更のためにプレーヤーのサイズ関数が呼び出されない限り、広告コンテンツのサイズは変更されません。他の方法（スタイルの幅と高さなど）を使用してプレーヤーのサイズを変更しても、広告のサイズは変更されません。

    使用する場合player.dimensions(width,height)プレーヤーのサイズを変更するには、トリガーする必要がありますfw-resizeplayerプラグインが寸法が変更されたことを認識できるようにするためのイベント。これは、一部のVPAID広告が広告のデフォルトのサイズを保持し、プレーヤーのサイズ変更時に自動的にサイズ変更されないためです。

    例を次に示します。

    player.dimensions(960,540);
player.trigger('fw-resizeplayer');

    オーバーレイ広告

    FreeWheelプラグインがFlashで広告をレンダリングしている場合、オーバーレイ広告はクリックできません。FreeWheelプラグインがHTML5で広告をレンダリングしている場合、オーバーレイは正しい座標で表示されません。HTML5オーバーレイ広告をクリックしても、クリックURLをたどってもコンテンツプレーヤーは一時停止しません。

    VPAID

    広告コントロールバーが有効になっている場合、VPAIDコントロールを介して広告を再開することはできません。
