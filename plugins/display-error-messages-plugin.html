    エラーメッセージプラグインの表示

    このトピックでは、デフォルトのエラーメッセージの動作を変更する方法、カスタムエラーメッセージを作成する方法、およびエラーをディスパッチする方法を学習します。

    概要

    エラーメッセージプラグインを使用すると、プレーヤーはエラーが発生したときにユーザーフレンドリーなメッセージを表示できます。デフォルトのスタイルシートは、ビデオ要素自体の上に半透明のオーバーレイとしてメッセージを表示します。既存のメッセージテキストを変更して、独自のスタイルを追加できます。必要なときにトリガーされるカスタムメッセージを作成することもできます。

    上の画像に示されているエラーメッセージは、無効なものでプレーヤーを更新することによって作成されましたsrcの値sourcesプロパティ。

    エラーメッセージプラグインはデフォルトのプラグインであり、BrightcovePlayerに自動的にロードされます。ただし、ロードしないことを選択できます。このプラグインがないと、限られたエラーメッセージのセットが表示され、一部のエラーはブラウザコンソールにのみ表示されます。プレーヤーの作成時にデフォルトのプラグインをロードしない方法の詳細については、プレーヤープラグインの概要資料。

    エラーメッセージをカスタマイズするには、このドキュメントに示されている2つの方法のいずれかを使用できます。
    • ページにコードを配置します。これにより、ページ上のプレーヤーのみが変更されます。
    • カスタムプラグインを使用する：これにより、プレーヤーのすべてのインスタンスが変更されます。
    プレーヤーの構成を変更してエラーをカスタマイズすることはできません。

    はじめに

    プレーヤーのすべてのインスタンスを更新するには、StudioのBrightcovePlayersモジュールを使用してカスタムプラグインを実装できます。このアプローチは、プレーヤーのエラーメッセージプラグインを更新するために次のセクションで使用されます。コードページからこのプラグインを更新することを選択した場合、それはプレーヤーのそのインスタンスにのみ影響します。

    標準エラー

    このプラグインには、ランタイムエラーコード値に基づいた標準のHTML5ビデオエラーのデフォルトのエラーメッセージのセットがあります。

    • エラーコード:1
      • タイプ :MEDIA_ERR_ABORTED
      • 見出し :ビデオのダウンロードがキャンセルされました
      • メッセージ :メディアの再生を中止しました
    • エラーコード:2
      • タイプ :MEDIA_ERR_NETWORK
      • 見出し :ビデオ接続が切断されました。インターネットに接続していることを確認してください
      • メッセージ :ネットワークエラーが原因で、メディアのダウンロードが途中で失敗しました。現在、MP4および/またはプログレッシブダウンロードのビデオフォーマットに最も役立ちます。を参照してください既知の問題点詳細については、このドキュメントのセクションを参照してください。
    • エラーコード:3
      • タイプ :MEDIA_ERR_DECODE
      • 見出し :動画が悪い、またはブラウザで再生できない形式になっている
      • メッセージ :破損の問題または使用しているメディアがブラウザでサポートされていない機能のため、メディアの再生が中止されました。
    • エラーコード:4
      • タイプ :MEDIA_ERR_SRC_NOT_SUPPORTED
      • 見出し :この動画は利用できないか、このブラウザによってサポートされていません
      • メッセージ :サーバーまたはネットワークのエラーのため、またはフォーマットがサポートされていないため、メディアをロードできませんでした。
    • エラーコード:5
      • タイプ :MEDIA_ERR_ENCRYPTED
      • 見出し :視聴しようとしている動画は暗号化されており、復号化方法がわかりません
      • メッセージ :メディアは暗号化されており、復号化するためのキーがありません。

    エラーに関連するエラーコードがない場合は、一般的なメッセージが表示されます。

    • エラーコード：不明
      • メッセージ:MEDIA_ERR_UNKNOWN
      • 説明：予期しない問題が発生しました。しばらくしてからもう一度確認して、再試行してください。

    テキストを上書きする

    変更できるエラーメッセージには、次の3つの部分があります。

    • headline :これは上部のメッセージテキストです。
    • type :これはエラーコード：テキスト。
    • message :これは技術的な詳細：テキスト。

    以下の例は、標準エラーのメッセージテキストをエラーコード値で上書きする方法を示しています。4。プロパティは次のように定義されています。

    • plugins:このプロパティには、プロパティと値の配列が含まれています。このプラグインの場合、必要なのはnameの値を持つプロパティerrors。
    • options:このプロパティは、プラグインにデータを渡すために使用されます。
    • errors:このプロパティは、更新するエラーコードを定義します。ここでは、のメッセージテキストを上書きしていますheadline、type、およびmessage。
      注：エラーコードは、テキストの上書きなど、任意の文字列値にすることができます。unknownメッセージ。

    ページコードでの使用

    コードにエラースクリプトを含めると、次のようにメッセージテキストを上書きできます。

    myPlayer.errors({
  "errors": {
    "4": {
      "headline": "This is a custom error message",
      "type": "custom type",
      "message": "these are details"
    }
  }

    カスタムプラグインの使用

    プレーヤーのすべてのインスタンスを更新する場合は、カスタムプラグインを作成し、Video CloudStudioのプレーヤーモジュールでプレーヤーに追加します。プラグインの詳細については、プレーヤープラグインの構成資料。

    標準のメッセージテキストを上書きするプラグインを作成するには、次の手順に従います。

    1. Brightcove Playerプラグインコードを使用して、インターネットでアクセス可能な場所に保存されたJavaScriptファイルを作成します。これに似ているはずですが、独自の値があります。 

      videojs.registerPlugin('errorText', function() {
  var myPlayer = this;

  myPlayer.errors({
    "errors": {
      "4": {
        "headline": "The Live Stream will begin soon",
        "type": "CUSTOM_TYPE",
        "message": "Please come back, once the live event has begun!"
      }
    }
  });
});

    2. の中にプレイヤーモジュール、選択プラグイン左のナビゲーションから。

    3. を展開しますプラグインを追加するボタンをクリックし、カスタムプラグイン。

    4. プラグインの詳細ページで、次の値を追加します。

      • プラグイン名 -プラグイン名
      • JavaScriptのURL -最初のステップからのプレーヤープラグインの場所
    5. [ 保存 ] ボタンを選択します。

    6. プレーヤーを公開します。

      プラグインコードに変更を加えるときは、それらの変更を取得するためにプレーヤーを公開する必要があります。

    Brightcoveが定義したカスタムエラー

    カスタムエラーも定義できます。Brightcoveは、このセクションで説明するいくつかのカスタムエラーを定義しました。また、次のセクションで詳しく説明するカスタムエラーを作成することもできます。

    • Brightcoveでは、カスタムエラーコード値を文字列にすることをお勧めします。提供されたエラーのうち2つは、歴史的な理由で負の数を使用していることがわかりますが、英数字/説明文字列が衝突の問題を引き起こす可能性は低くなります。
    • カスタムエラーメッセージには、任意の名前を付けることができます。Brightcoveは、タイプがで始まることをお勧めしますPLAYER_ERR対標準化MEDIA_ERR混乱を避けるために。
    • 詳細にこのセクションの後半、カスタムエラーを却下できるかどうかを指定できます。

    5つのカスタムエラーメッセージが参照として追加されました。

    • エラーコード：-1
      • メッセージ:PLAYER_ERR_NO_SRC
      • 説明：動画がロードされませんでした
    • エラーコード：-2
      • メッセージ:PLAYER_ERR_TIMEOUT
      • 説明：動画をダウンロードできませんでした
        注：一般に、PLAYER_ERR_TIMEOUTは、プレーヤーがビデオを再生することを期待しているが、 45秒間進行できません。これは、たとえば、タブがバックグラウンドに配置されたときにブラウザが停止するなど、以前に発生した多くのことの最終結果になる可能性があります。ただし、再生を続行できないネットワークまたはプラットフォームの状態の一種のキャッチオール状態になる可能性があります。
    • エラーコード:設定されていません
      • メッセージ:PLAYER_ERR_DOMAIN_RESTRICTED
      • 説明：この動画は現在のドメインでの再生が制限されています
    • エラーコード:設定されていません
      • メッセージ:PLAYER_ERR_IP_RESTRICTED
      • 説明：このビデオは現在のIPアドレスで制限されています
    • エラーコード:設定されていません
      • メッセージ:PLAYER_ERR_GEO_RESTRICTED
      • 説明：この動画は、現在の地域での再生が制限されています
    コードのないエラーの場合、メッセージがコードに使用されます。たとえば、ドメイン制限エラーは次のように表示されます。

    ユーザー定義のカスタムエラー

    独自のカスタムエラーを定義する場合、Brightcoveは使用しないことをお勧めしますコード。(You see in the section above that this is what Brightcove is now doing with the newer custom errors they are defining.)また、使用を検討する必要がありますPLAYER_ERR_一貫性を保つためにカスタムエラーのプレフィックスを付けますが、もちろん、好きなように名前を付けることができます。

    エラーをローカライズする

    エラープラグインは複数の言語をサポートしています。言語サポートを追加するには、プラグインがロードされた後、使用する特定の言語ファイルをロードします。

    <script src="lang/es.js"></script>

    その後、に示されている手法を使用できます。Brightcove ブプレーヤーをプログラムでローカライズするエラーメッセージをローカライズするためのドキュメント。

    bc-catalog-error

    ready()スクリプトブロックの通常のセクションでエラーを処理すると、問題が発生する可能性があります。たとえば、プレーヤーの準備が整う前にbc-catalog-errorイベントがディスパッチされる可能性があります。また、エラーをリッスンすると、ready()セクションでは、エラーを処理できなくなります。この問題は、ジオフィルタリングを使用している場合、ビデオが非アクティブである場合、ビデオがスケジュール範囲外である場合、または別のアカウントにある場合に発生する可能性があります。コードに問題はないかもしれませんが、問題はブラウザに依存する可能性がありますので、注意してください。

    たとえば、ビデオが非アクティブのときにメッセージを表示するプラグインコードは次のとおりです。

    videojs.registerPlugin('inactiveErrorCheck', function() {
  var myPlayer = this;
  myPlayer.one('bc-catalog-error', function(){
    var specificError;
    myPlayer.errors({
      'errors': {
          'inactive-video-error': {
          'headline': 'The video you are trying to watch is inactive.',
          'dismiss': false
        }
      }
    });
    if (typeof(myPlayer.catalog.error) !== 'undefined') {
      specificError = myPlayer.catalog.error.data[0];
      if (specificError !== 'undefined' & specificError.error_code == "VIDEO_NOT_PLAYABLE") {
        myPlayer.error({code:'inactive-video-error'});
      };
    };
  });
});
    カタログエラーの処理の詳細については、 bc-catalog-errorを処理するのセクションプレイヤーカタログ資料。

    誇張されたエラー率

    報告されたエラーの数が不当に多いように見える場合は、同じセッションで重複したエラーイベントが発生し、この誇張されたエラー率が発生する可能性があることを知っておくことが重要です。Brightcove Playerは、エラーがプレーヤーに報告されるのと同じ時間に、同じ量で分析にエラーを送信します。たとえば、プレーヤーにメディアがなく、コードが何らかの形で呼び出された場合play()千回続けて千回PLAYER_ERR_NO_SRCエラーはアナリティクスに報告されます。

    分析を歪めるエラーが大量にあるセッションがいくつかある場合は、実際の問題をよりよく理解するために、次のロジックの使用を検討する必要があります。

    number_of_sessions_with_errors / total_number_of_sessions
    のではなく 
    count of errors/number of views

    既知の問題

    • HLSソースを再生する場合、ネットワーク接続が失われた後の動作は、おそらく期待したものではありません。詳細は次のとおりです。
      • Brightcove Playerのバージョン6.xでは、HLSセグメントが際限なく要求され、 MEDIA_ERR_NETWORKこれまでに表示されます。
      • Brightcove Playerのバージョン5.xでは、一定期間（30秒以上）後PLAYER_ERR_TIMEOUTエラーが表示されます。
    • Windows 10のEdge（Studio内とパブリックURLの両方）にビデオをロードする場合、 MEDIA_ERR_SRC_NOT_SUPPORTEDエラーが表示され、ビデオを再生できません。

    • AndroidデバイスとiPhoneでは、エラーメッセージのタップイベントが親のビデオ要素にバブルアップしません。つまり、表示されたエラーメッセージを閉じることができません。ユーザーがフルスクリーンモードの場合、この状態を終了する方法がないため、この動作が問題になる可能性があります。

      この問題は現在作業中であり、将来のプレーヤーリリースで修正される予定です。今のところ、次のような回避策を試すことができます。

      player.on("touchstart", function(e) {
  if (player.error_) {
    player.error(null);
    e.preventDefault();
  }
})
