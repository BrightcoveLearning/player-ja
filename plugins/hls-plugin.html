    HLS プラグイン

    HLSプラグインはBrightcovePlayerプラグインであり、W3CのMedia Source Extensions（MSE）に依存して、ネイティブでサポートされていないプラットフォームでHTTP Live Streaming（HLS）ビデオを再生します。HLSプラグインはプレーヤーに自動的に含まれます。

    HLSプラグインの基本

    次の点は、HLSプラグインを理解して使用するのに役立ちます。

    • このドキュメントの冒頭の文で述べたように、プラグインはW3CのMedia Source Extensions（MSE）に依存しています。MSEは、JavaScriptがHTML5ビデオをサポートするWebブラウザ内のメディアコーデックにバイトストリームを送信できるようにするW3C仕様です。他の可能な用途の中でも、これにより、ストリーミングメディア用のクライアント側のプリフェッチおよびバッファリングコードを完全にJavaScriptで実装できます。
    • プラグインを使用すると、プレーヤーでHLS（m3u8）ビデオコンテンツを使用できます。たとえば、メディアセクションに次の構成を使用してプレーヤーを作成できます。 
      "media":{
    "sources": [{
        "src": "http://example.com/video.m3u8",
        "type": "application/x-mpegURL"
    }]
}
    • HLSを使用する場合、クロスオリジンリソースシェアリング（CORS）が問題になる可能性があります。CORSの使用の詳細については、 CORSガイド。
    • HLSは、IE11より前のバージョンのIEではサポートされていません。

    概要

    HTTPライブストリーミング（HLS）は、iOSおよびAndroidでのネイティブサポートのおかげで、モバイルデバイスでビデオをストリーミングするためのデファクトスタンダードになりました。ただし、プラットフォームに関係なく、フォーマットを推奨する理由はいくつかあります。

    • （クライアント駆動の）アダプティブビットレート選択をサポート
    • 標準HTTPポート経由で配信
    • シンプルなテキストベースのマニフェスト形式
    • 独自のストリーミングサーバは不要

    残念ながら、Safariを除くすべての主要なデスクトップブラウザにはHLSサポートがありません。そのため、Web開発者は、同じビデオの代替レンディションを維持する必要があり、最高のデスクトップ表示エクスペリエンスを提供するためにHTMLベースのビデオを完全に放棄しなければならないという不幸な立場に置かれています。

    このプラグインは、Media Source ExtensionsまたはFlashをサポートするブラウザーでHLSのポリフィルを提供することにより、この状況に対処します。単一のHLSストリームをデプロイし、通常のHTML5ビデオAPIに対してコードを記述し、すべての大きなWebデバイスカテゴリにわたって高速で高品質のビデオエクスペリエンスを作成できます。

    現在、現時点では番号へのサポート：

    • 代替オーディオおよびビデオトラック
    • セグメントコーデック以外 AACオーディオ付きH.264
    • Internet Explorer <11
    注：すべてのMSE実装との互換性を最大にするために、Brightcoveは設定をお勧めしますaudio_bitrate少なくとも48取り込みプロファイルで。

    オプション

    HLSプラグインを構成するために使用できるいくつかのオプションがあります。

    withCredentials

    タイプ: boolean

    次のように使用できます。

    • ソースオプション
    • 初期化オプション

    いつwithCredentialsプロパティはに設定されていますtrue、マニフェストとセグメントに対するすべてのXHRリクエストにはwithCredentialsに設定true同様に。これにより、マニフェストとセグメントが存在するサーバーからCookieを保存して渡すことができます。設定すると、CORSに影響があります。Access-Control-Allow-Originヘッダーをに設定することはできません*、また、応答ヘッダーには次の追加が必要ですAccess-Control-Allow-Credentialsに設定されているヘッダーtrue。を参照してくださいHTML5Rocksの記事詳細については。

    既知の問題：withCredentialsデスクトップまたはデバイス上のSafariでは機能しません。

    プラグインは、プレーヤー管理API HTTPを使用するPATCHここに示すように、メソッド：

    --header "Content-Type: application/json" \
    --user YOUR_EMAIL \
    --request PATCH \
    --data '{ "hls": { "withCredentials": true } }' \
https://players.api.brightcove.com/v2/accounts/YOUR_ACCOUNT_ID/players/YOUR_PLAYER_ID/configuration

    を設定することもできますwithCredentials上記のようにプレーヤーベースではなく、ソースごとのオプション。たとえば、ソースを設定するときに含めることができますwithCredentials、ここに示すように：

    curl \
  --header "Content-Type: application/json" \
  --user $EMAIL \

  --data '{
      "name": "MySamplePlayer",
      "configuration": {
        "media": {
          "sources": [{
            "src":"http://solutions.brightcove.com/bcls/assets/videos/Tiger.mp4",
            "type":"video/mp4",
            "withCredentials": true

      }
    }' \
https://players.api.brightcove.com/v2/accounts/$ACCOUNT_ID/players

    ランタイムプロパティ

    一般に、HLSオブジェクトには次の方法でアクセスできます。

    • Brightcove Player v5：player.hls
    • Brightcove Player v6：player.tech().hls
      呼び出すことに注意してくださいplayer.tech()上級ユーザーにのみ推奨されるため、コンソールに警告を記録します。合格できますplayer.tech(true)警告を抑制します。

    player.hls.playlists.master

    タイプ: object

    解析されたマスタープレイリストを表すオブジェクト。メディアプレイリストを直接ロードすると、エントリが1つだけのマスタープレイリストが作成されます。

    player.hls.playlists.media

    タイプ: function

    現在アクティブなメディアプレイリストを取得または変更するために使用できる機能。アクティブなメディアプレイリストは、追加のビデオデータをダウンロードする必要があるときに参照されます。引数なしでこの関数を呼び出すと、アクティブなメディアプレイリストの解析済みプレイリストオブジェクトが返されます。マスタープレイリストのプレイリストオブジェクトまたはマスタープレイリストで指定されたURI文字列を使用してこの関数を呼び出すと、指定されたメディアプレイリストの非同期ロードが開始されます。取得されると、アクティブなメディアプレイリストになります。

    player.hls.bandwidth

    タイプ: number

    最後のセグメントのダウンロードで1秒あたりにダウンロードされたビット数。この値は、のデフォルトの実装で使用されますselectPlaylist再生する適切なビットレートを選択します。最初のビデオセグメントがダウンロードされる前は、帯域幅を正確に見積もることは困難です。HLS技術は、プレイリストのダウンロード時間に基づくヒューリスティックを使用して、デフォルトでこの見積もりを行います。より正確な帯域幅情報のソースがある場合は、HLS技術がロードされたらすぐにこの値をオーバーライドして、初期の帯域幅の見積もりを提供できます。

    player.hls.stats.bytesReceived

    タイプ: number

    HLS技術によってダウンロードされたコンテンツバイトの総数。

    player.hls.selectPlaylist

    タイプ: function

    次のセグメントのダウンロードに使用するメディアプレイリストオブジェクトを返す関数。これは、新しいセグメントがダウンロードされる直前にプラグインによって呼び出されます。この関数をオーバーライドして、アダプティブストリーミングロジックを提供できます。ただし、に存在する有効なメディアプレイリストオブジェクトを必ず返す必要があります。player.hls.playlists.master。

    イベント

    ロードされたメタデータ

    ストリームの最初のメディアプレイリストがダウンロードされた後に発生します。

    ロードされたプレイリスト

    新しいマスターまたはメディアプレイリストがダウンロードされた直後に発生します。デフォルトでは、プラグインは必要に応じてプレイリストのみをダウンロードします。

    mediachange

    新しいプレイリストがアクティブなメディアプレイリストになると発生します。実際のレンダリング品質の変更は、このイベントと同時に発生しないことに注意してください。新しいセグメントを要求し、既存のバッファーを最初に使い果たす必要があります。

    エラー時にソースをリロード

    HLSプラグインを使用する場合、プレーヤーによってエラーが発生した現在の時刻にソースをリロードする呼び出し可能なメソッドがあります。この機能をオンにするには、を呼び出す必要がありますreloadSourceOnError()方法。次の短いビデオは、実際のメソッドを示しています。ビデオに示されているすべてのコードについては、このセクションの後半で説明します。

    注：元々はライブストリーミングを目的としていましたが、reloadSourceOnError()ビデオに示されているように、ビデオオンデマンドでも機能します。

    の構文reloadSourceOnError()方法は次のとおりです。

    reloadSourceOnError(optionsObject)

    オプションoptionsObject次のプロパティがあります。

    プロパティ データ・タイプ デフォルト値 説明
    errorInterval 数値 30 リロードがトリガーされるために2つのエラー間で経過する必要がある最小時間（秒単位）。たとえば、時間を10に設定すると、エラーが発生するたびに、関数は10秒未満前にリロードが発生したかどうかを確認します。時間間隔未満が経過した場合、ソースはリロードされません。(This is to ensure that content with an error doesn't reload constantly.)指定された間隔よりも長い時間が経過した場合、エラーが発生した時点でビデオが再ロードされます。
    getSource() 機能 現在のソースを取得します ソースオブジェクトをロードまたはリロードするために呼び出される関数。デフォルトでは、プレーヤーの現在のソースを取得します。

    アダプティブスイッチング

    HLSレンディションの選択

    再生中、プレーヤーはアルゴリズムに基づいてより高いまたはより低いレンディションに切り替わります。このアルゴリズムへの入力は次のとおりです。

    • 使用可能な帯域幅
    • プレーヤーの寸法

    レンディションの選択の詳細については、どのレンディションが再生されるかを決定する資料。

    MP4レンディションの選択

    何らかの理由でBrightcovePlayerがHLSソースを再生できない場合、MP4の再生にフォールバックします。この場合、モバイルデバイスでビデオを表示してMP4を再生すると、プレーヤーはビットレートが0.5 MB / sに最も近いMP4を選択します。デスクトップまたはラップトップデバイスの場合、3MB /秒に最も近いMP4レンディションが選択されます。

    インバンドメタデータ

    Brightcove Playerは、HLSビデオストリームに埋め込まれた特定のタイプのID3タグ情報を認識します。ID3標準は、もともとMP3オーディオトラックに関するメタデータを提供するために使用されていました。(The acronym is from IDentify MP3.)ストリームにメタデータが埋め込まれている場合、インバンドメタデータテキストトラックが自動的に作成され、ストリームで検出されたときにキューが入力されます。一般的な使用例は、広告をライブストリームに表示する必要がある場合にID3データが指示することです。

    ID3標準では多くのフレームタイプが定義されていますが、次の2つのUTF-8エンコードフレームのみがキューポイントにマップされ、それらの値がキューテキストとして設定されます。

    • WXXX-ユーザー定義のURLリンクフレーム
    • TXXX-ユーザー定義のテキスト情報フレーム

    他のすべてのフレームタイプに対してキューが作成され、データは生成されたキューに添付されます。

    608キャプション

    BrightcoveのHLSプラグインは、608キャプションをサポートしています。CEA-608キャプション、EIA-608キャプション、およびLine 21キャプションとも呼ばれる608キャプションは、米国およびカナダでのNTSCTVアナログ放送のクローズドキャプションの標準です。608のキャプションは、ライブストリームに挿入して、HLSにミックスすることができます。 ts（トランスポートストリーム）ファイル。

    ホスティングの問題

    ネイティブHLS実装とは異なり、HLSプラグインはブラウザのセキュリティポリシーに準拠する必要があります。つまり、ストリームを構成するすべてのファイルは、ビデオプレーヤーをホストしているページと同じドメインから、または適切なサーバーから提供される必要があります。 CORSヘッダー構成済み。簡単指示が利用可能です人気のあるウェブサーバーやほとんどのCDNの場合、アカウントでCORSをオンにするのに問題はありません。

    エラー

    HLS再生中のエラーは、タイプを使用して報告されますAPPEND_BUFFER_ERR。メッセージは、ブラウザのネイティブエラーから取得されたものになります。例えば、割り当てを超えました。
