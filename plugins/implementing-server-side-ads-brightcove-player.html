    ブライトコーブプレーヤーでのサーバーサイド広告の実装

    このトピックでは、Brightcove Playerでサーバーサイド広告挿入（SSAI）を使用して、ビデオストリームにステッチされた広告を配信する方法を学習します。

    概要

    このサンプルおよび関連するコードは、本番環境の開発のためのガイドとして提供されます。このコードは、実稼働での使用が認定されておらず、公式にはサポートされていません。

    サーバーサイド広告挿入（SSAI）を使用すると、動画に広告を埋め込むことで、ブラウザーの広告ブロッカーによってブロックされないようにすることができます。動的配信は、ストレージフットプリントを削減し、メディアを動的にパッケージ化する次世代の取り込みおよび配信システムです。

    デフォルトでは、SSAIはすべての広告が監視されるように強制し、再生中に広告カウントダウンタイマーを表示します。この機能を簡単にカスタマイズして、広告をスキップできます。

    Brightcove Playerは、ブロックされていないときにクライアント側の広告を使用し、広告ブロッカーが検出されると自動的にSSAIにフェイルオーバーするように構成できます。この機能を有効にする方法の詳細については、広告フェイルオーバー資料。

    注

    • SSAI を使用するには、Video Cloud アカウントをダイナミック配信に設定し、SSAI を有効にする必要があります。この機能の使用を開始するには、アカウントマネージャーにお問い合わせください。
    • ライブ再生用のSSAIはSSAIプラグインを必要とせず、SSAI再生用のクライアント側機能はライブでは使用できません。詳細については、 SSAIを使用したBrightcoveLive API資料。
    • 制限事項のリストについては、「 SSAI の概要」ドキュメントを参照してください。

    ビデオの概要については、以下を参照してください。

    プレーヤーの例

    この例では、VMAPファイルで定義されたIMA広告を使用して、ビデオストリームでサーバー側の広告を配信します。プレロール、ミッドロール、ポストロールの広告があることがわかります。VMAPファイルは広告設定で定義されています。

    ペンを見る18468-advertising-ssai-plugin Brightcove Learning Services（ @ bcls1969） オンCodePen。

    ソースコードを表示します。

    機能

    Brightcove SSAIは、OnceUX広告配信の最新の代替品です。機能は次のとおりです。

    • より完全なVMAP / VAST解析
    • VASTコンパニオン広告の完全サポート
    • タイムラインと線形広告ロールを操作するための新しいAPI
    • プレイリスト、広告マクロ、FairPlayのサポート
    • SSAI は DRM コンテンツと DRM 以外のコンテンツの両方で動作します。
    • SSAIの主な役割の1つは、シークと広告に関する特定の動作を強制することです。
    • 従来のOnceUX VMAP（uo名前空間を使用）と新しいDynamic Delivery VMAP（bc名前空間を使用）のデュアルサポート

    入門編

    Video Cloudからサーバー側の広告を再生するには、次の手順に従います。

    1. 広告設定を作成する
    2. ブライトコーブプレーヤーを作成する
    3. Studioを使用してSSAIを実装する

    それだこれで、BrightcovePlayerがサーバー側広告用に構成されました。必要に応じて、に示すように、プログラムでSSAIを追加できます。プログラムでSSAIを実装するセクション。

    広告設定を作成する

    広告設定では、広告コール、ビーコン、その他の設定オプションなど、SSAI 再生のさまざまな側面を定義します。広告レスポンスには、VAST、VMAP、DFP広告ルールがあります。広告設定を作成するには、次の手順に従います。

    1. Video Cloud Studio で、[ 管理 ] メニューを展開し、[ サーバーサイド広告設定 ] を選択します。

    2. 広告設定の情報を追加して、[ 保存 ] を選択します。

      3. https://studio.support.brightcove.com/admin/configuring-server-side-ad-settings.html

    詳細については、「サーバーサイド広告設定の構成」を参照してください。

    SSAI API を使用して広告設定を作成するには、 Video Cloud SSAI 広告設定 API ドキュメントを参照してください。

    Brightcoveプレーヤーを作成する

    Video CloudStudioを使用して新しいBrightcoveプレーヤーを作成します。詳細については、ビデオクラウドの基本：プレーヤーの作成資料。

    バージョン 6.8.0 以降の Brightcove Player を使用する必要があります。

    Studioを使用したSSAIの実装

    サーバーサイド広告用にプレーヤーを構成する最も簡単な方法は、Video CloudStudioを使用することです。広告構成とプレーヤーを作成したら、次のようにSSAI用にプレーヤーを構成する準備が整います。

    1. PLAYERS モジュールを開き、広告機能を追加するプレーヤーを見つけます。
    2. プレーヤーのリンクを選択して、プレイヤーのプロパティを開きます。
    3. 左側のナビゲーションメニューで [ 広告 ] を選択します。
    4. ［サーバー側（SSAI）を有効にする］チェックボックスをオンにします。
    5. [ 設定の選択 ] ドロップダウンメニューから、このプレーヤーに関連付ける広告設定を選択します。

    6. オーバーレイを広告の上に表示する場合は、[ 広告情報オーバーレイを有効にする ] チェックボックスをオンにします。これには、「詳細」と広告カウントダウンオーバーレイが含まれます。

      ここでオーバーレイを有効にし、VAST 広告タグで設定するまで、オーバーレイは表示されません。

    7. 完成したフォームは次のようになります。

    8. [ 保存 ] を選択します。
    9. プレーヤーを公開するには、[ パブリッシュと埋め込み] [> 変更を公開 ] を選択します。

    広告プロパティへの変更が保存されると、SSAIプラグインはプラグイン設定の一部として構成されます。JavaScriptとCSSは、を介して追加したため非表示になります広告セクション。

    広告で動画を再生する

    動的配信で取り込んだ動画には、VMAP ファイルで指定された広告が広告設定に含まれます。SSAI が動作するためには、ビデオにオーディオトラックが関連付けられている必要があります。

    プログラムによるSSAIの実装

    前のセクションで示したように、Video CloudStudioを使用してSSAIをプレーヤーに簡単に追加できます。これをプログラムで実行する場合は、次の手順に従ってください。

    1. SSAIプラグインを追加します
    2. 広告をプレーヤーに関連付ける
    3. 広告で動画を再生する

    SSAIプラグインを追加します

    SSAIプラグインファイルをHTMLコードに追加するか、次のようにプレーヤーの構成に追加することができます。

    1. PLAYERS モジュールを開き、新しいプレーヤーを作成するか、プラグインを追加するプレーヤーを見つけます。
    2. プレーヤーのリンクをクリックして、プレイヤーのプロパティを開きます。
    3. 左側のナビゲーションメニューで [ プラグイン ] をクリックします。
    4. 次に、3 つのオプションから [ プラグイン ] をクリックします。
    5. [ プラグインの追加 ] から [ カスタムプラグイン ] を選択します。
    6. [ プラグイン名 ] に、と入力しますssai。
    7. JavaScript の URL には、次のように入力します。 
      https://players.brightcove.net/videojs-ssai/1/videojs-ssai.js
    8. CSS の URL には、次のように入力します。 
      https://players.brightcove.net/videojs-ssai/1/videojs-ssai.css
    9. このプラグインのオプションを入力する必要はありません。
    10. [保存]をクリックします。
    11. プレーヤーを公開するには、[ パブリッシュと埋め込み] [> 変更を公開 ] をクリックします。
    12. 開くダイアログを閉じるには、[ 閉じる ] をクリックします。

    広告をプレーヤーに関連付ける

    次に、いずれかの広告構成をBrightcovePlayerに関連付けます。これは、次の3つの方法のいずれかで実行できます。

    • 標準の埋め込みコード
    • 高度な埋め込みコード
    • カタログの使用
    • プレーヤー構成の更新

    標準 (iframe) 埋め込みコード

    標準の埋め込みコードに、adConfigId広告設定IDの値を使用したクエリ文字列パラメータ：

    <iframe src="https://players.brightcove.net/1752604059001/default_default/
index.html?videoId=5625751316001&adConfigId=your ad config id"
	allowfullscreen
	webkitallowfullscreen
	mozallowfullscreen
	width="640"
	height="360"></iframe>

    高度な埋め込みコード

    Advanced embedコードを使用して、data-ad-config-id広告設定IDの値を持つ属性：

    <video-js data-video-id="5625751316001"
  data-account="1752604059001"
  data-player="default"
  data-embed="default"
  data-application-id
  data-ad-config-id="your ad config id"
  class="video-js"
  controls
  width="640"
  height="360"></video-js>
<script src="https://players.brightcove.net/1752604059001/default_default/index.min.js"></script>

    カタログの使用

    あなたは使用することができますplayer catalog広告を動画に関連付けるため。カタログは、次の2つの要求を行うことで機能します。

    1. プレーヤーカタログからビデオデータを要求します。これにはVMAPURLが含まれます。
    2. プレーヤーのソースはVMAPURLで設定され、DynamicDeliveryからのVMAPドキュメントの要求をトリガーします。プレーヤーソースは、有効なVMAPXMLドキュメントで再度設定されます。

    広告付きの動画を再生する

    動的配信で取り込んだ動画には、VMAP ファイルで指定された広告が広告設定に含まれます。SSAI が動作するためには、ビデオにオーディオトラックが関連付けられている必要があります。

    オプション

    • debug
      • trueの場合、contrib-adsにデバッグメッセージを設定し、videojs-bc-analytics-loggerの存在下で追加情報をログに記録します。
    • hideOverlays
      • trueの場合、カウントダウンタイマーともっと詳しく知る広告の再生中は、クリックスルーオーバーレイは表示されません。
    • trackingBeacons
      • falseの場合、広告ビュー、インプレッション、四分位数などについてVMAPから解析されたトラッキングビーコンは送信されません。
    • timeout
      • VMAPをフェッチするXHRがタイムアウトするまでのミリ秒数。

    スタイル設定

    このプラグインによってプレーヤーに適用される便利なHTMLクラスがいくつかあり、プラグインの状態を判別するためにターゲットにすることができます。

    クラス 用途
    vjs-ssai SSAIプラグインがインスタンス化されているが、必ずしも有効になっていないことを示します。これは、SSAIソースを再生していない場合でも存在します。
    vjs-ssai-enabled SSAIプラグインは現在有効になっています。つまり、SSAIソースがプレーヤーに設定されています。
    vjs-ssai-disabled SSAIプラグインは現在有効になっていません。
    vjs-ssai-waiting SSAIプラグインは、データまたはその他の外部プロセスを待機しています。
    vjs-ssai-not-waiting SSAIプラグインは何も待っていません。
    vjs-ssai-hide-overlays ザ・hideOverlaysオプションがに設定されていますtrue。
    vjs-ssai-show-overlays ザ・showOverlaysオプションがに設定されているtrue。これがデフォルトです。

    タイムライン状態メソッド

    ザ・*TimelineStateメソッドはTimelineStateオブジェクト。タイムライン状態オブジェクトは、絶対タイムラインの任意の時点でのSSAIストリームの再生状態を詳細に示す特定のインターフェイスを備えたプレーンオブジェクトです。

    「絶対時間」とは、ストリームの完全なタイムライン内のポイントを指します（広告とコンテンツの両方が含まれます）。「相対時間」という用語は、現在のコンテンツ（広告またはコンテンツのいずれか）に相対的な時間を指します。

    contentTimelineState()

    • 引数:単一の数値time
    • 指定されたコンテンツ時間のタイムライン状態を取得します（コンテンツ、広告はカウントされません）。たとえば、コンテンツの長さが100秒の場合、次のコマンドを使用して、コンテンツの途中でタイムラインの状態を取得できます。 
      contentTimelineState(50)

    absoluteTimelineState()

    • 引数:単一の数値time
    • 指定された絶対時間でのタイムライン状態を取得します（コンテンツ+広告）。たとえば、コンテンツに30秒のプレロール広告が含まれている場合、次の方法を使用して、コンテンツの30秒でタイムラインの状態を取得できます。 
      absoluteTimelineState(50)

    追加のタイムライン状態メソッドのリストは次のとおりです。

    currentTimelineState()
    relativeTimelineState()
    seekInAbsoluteTime()
    seekInRelativeTime()
    seekInContentTime()

    構成に関する注意事項

    1. 広告のプリロードはSSAIで行うべきではありません。この理由は、プレロードすると、プレーヤーが広告インプレッションを報告し、おそらく動画が再生される前の最初の四分位ビーコンを報告するからです。これにより、広告分析が不正確になる可能性があります。StudioでSSAIを構成すると、これは自動的に行われますが、SSAIを手動でセットアップする場合は、この問題に注意する必要があります。
    2. ウェブプレーヤーが SSAI を使用していて、その動機の 1 つが広告ブロッカーを回避することである場合は、サーバー側のビーコンを使用する必要があります。クライアント側のビーコンはブロックされるため使用しないでください。

    用語集

    このプラグインは、の概念を区別します絶対のそしてコンテンツ SSAIストリーム内の時間。従来のビデオプレーヤーには、コンテンツ時間;現在再生中のURIの開始から終了までの時間。SSAIストリームは本質的に多数であるためコンテンツストリームをつなぎ合わせて、の概念を導入しました絶対のビデオ広告を含む完全なステッチストリームを考慮に入れた時間。

    プレフィックスが表示されたら絶対のプロパティまたはメソッドでは、予想/返される時間は、ステッチされたストリーム全体を基準にしています。プレフィックスが表示されたらコンテンツ、予想/返される時間は、ストリームにステッチされた特定のコンテンツ（メインコンテンツまたは単一の線形広告）のみに関連しています。

    • 絶対時間：SSAIストリームの合計タイムライン内の任意の時点を指します。たとえば、 2:00とのビデオ0:30プレロール広告の絶対時間の合計は2:30。の絶対時間0:15プレロールにあり、の絶対時間0:31コンテンツの最初の1秒です。

    • 相対時間：メディアの現在のブロック（コンテンツまたは広告）に関連する時間を指します。上記を拡張して、プレロール中の相対時間0:15の絶対時間と同義です0:15、しかし絶対時間0:31の相対時間に相当します0:01。

      一般に、相対時間はプレーヤーUIに表示されるものであり、このプラグインと関連するミドルウェアの仕事の多くは絶対時間から相対時間に変換されます。

    • コンテンツ時間 SSAIストリームのコンテンツタイムライン内のある時点を指します-すべての広告を無視します。たとえば、0：30のプレロール広告を含む2:00の動画のコンテンツ時間は2:00です。0:15のコンテンツ時間は、0：45の絶対時間（プレロールプラス15秒）に相当します。

    既知の問題

    SSAIプラグインを使用する際の既知の問題は次のとおりです。

    • Safari 10/11は、ビデオの最後にポストロールの最後のフレームを表示することがあります。
    • SSAIは、オーバーレイ広告をビデオストリームにつなぎ合わせません。
