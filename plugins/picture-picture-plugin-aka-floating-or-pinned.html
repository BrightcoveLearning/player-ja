    ピクチャインピクチャプラグイン (別名「フローティング」または「固定」)

    このトピックでは、ピクチャーインピクチャープラグインを使用して、Brightcoveプレーヤーでこのモードを有効にする方法を学習します。

    概要

    ピクチャーインピクチャーモードでは、ユーザーはマルチタスクを実行できます。ビデオの再生が開始されると、ユーザーがページを下にスクロールすると、ピクチャーインピクチャーモードが再配置され、プレーヤーがWebページの隅に固定されます。

    ピクチャーインピクチャープラグインは、Advanced（video） 埋め込みコード。標準では機能しません（iframe）実装。

    ピクチャーインピクチャーと呼ばれる理由

    このプラグインが有効にする動作は、一般に「フローティング」または「ピン留め」動作とも呼ばれます。ブライトコーブでは、AppleとGoogleがネイティブブラウザにこの表現を採用しているため、「ピクチャーインピクチャー」という用語を使用しています。W3C 基準「ピクチャーインピクチャー」も参照してください。

    プレーヤーの例

    ビデオの再生を開始します。(You can then stop it).ページを下にスクロールして、Webページの右隅にあるピクチャーインピクチャープレーヤーを確認します。ピクチャーインピクチャープレーヤーを閉じるか、フルサイズプレーヤーまでスクロールして戻ることができます。

    ペンを見るPicture-in-Pictureプラグイン Brightcove Learning Services（ @ bcls1969） オンCodePen。

    プレーヤーモジュールを使用して実装する

    ピクチャーインピクチャーを実装するには  Playersモジュールを使用するプラグインは、次の手順に従います。

    1. PLAYERS モジュールを開き、新しいプレーヤーを作成するか、プラグインを追加するプレーヤーを見つけます。
    2. プレーヤーのリンクを選択して、プレイヤーのプロパティを開きます。
    3. 左側のナビゲーションメニューで [ プラグイン ] を選択します。

    4. 次に、[ プラグインを追加 ] ボタンを選択し、[ Brightcove プラグイン ] を選択します。

    5. を展開しますBrightcoveプラグインドロップダウンして選択しますピクチャーインピクチャー。

    6. オプション :[ オプション (JSON) ] テキストボックスに構成オプションを入力します。ピクチャーインピクチャーモードのプレーヤーの水平方向の配置を設定する例を次に示します。

      を参照してくださいオプション詳細については、セクションを参照してください。

    7. [ 保存 ] ボタンを選択します。これで、ピクチャーインピクチャープレーヤーのプラグインのリストにプラグインが追加されました。

    8. プレーヤーを公開するには、[ パブリッシュと埋め込み] [> 変更を公開 ] を選択します。
    9. 開くダイアログを閉じるには、[ 閉じる ] を選択します。

    10. に戻るメディアモジュールを作成し、ピクチャーインピクチャー用に更新したプレーヤーを使用してビデオを公開します。

    11. エディターで、プレーヤーの埋め込みコードをWebページにコピーします。

    12. ピクチャーインピクチャープラグインでは、プレーヤーがクラスがに設定されたコンテナ要素でラップされている必要がありますvjs-pip-container。コードは次のようになります。

      <div class="vjs-pip-container">
  <video-js id="myPlayerID"
    data-video-id="5701202551001"
    data-account="1752604059001"
    data-player="default"
    data-embed="default"
    class="video-js"
    width="640" height="360"
    controls></video-js>
  <script src="https://players.brightcove.net/1752604059001/default_default/index.min.js"></script>
</div>

      <div>上記の要素がプレーヤーの親として見つからない場合、プラグインは初期化に失敗し、次のログ警告が表示されます。

      VIDEOJS: WARN: expected player to be a child of a "vjs-pip-container" element, cannot continue with picture-in-picture plugin
    13. ビデオの再生が開始されたら、下にスクロールして、ページの下部に固定されているピクチャーインピクチャープレーヤーを確認します。

    コードを使用して実装する

    カスタムコードを使用してプラグインを実装するには、次のプラグインプロパティを設定します。

    • scripts - JavaScript supplied for the plugin and will not change for different plugin implementations
    • stylesheets - CSS supplied to you for the plugin and will not change for different plugin implementations
    • plugin name - Always pip
    • plugin options - Contains an array of properties and values

    プラグインをコードに追加するには、次の手順を実行します。

    1. ピクチャーインピクチャープラグインのスタイルを追加します。 
      <link href="https://players.brightcove.net/videojs-pip/1/videojs-pip.css" rel="stylesheet">
    2. JavaScriptファイルを追加して、picture-in-pictureプラグインを含めます。 
      <video-js id="myPlayerID"
  data-video-id="5701202551001"
  data-account="1752604059001"
  data-player="default"
  data-embed="default"
  class="video-js"
  width="640" height="360"
  controls></video-js>
<script src="https://players.brightcove.net/1752604059001/default_default/index.min.js"></script>

<!-- Script for the picture-in-picture plugin -->
<script src="https://players.brightcove.net/videojs-pip/1/videojs-pip.min.js"></script>

    3. クラスをに設定して、プレーヤーの埋め込みコードをコンテナ要素にラップしますvjs-pip-container。ピクチャーインピクチャープラグインでは、プレーヤーがこのコンテナ要素でラップされている必要があります。

      <div class="vjs-pip-container">
    <video-js id="myPlayerID"
      data-video-id="5701202551001"
      data-account="1752604059001"
      data-player="default"
      data-embed="default"
      class="video-js"
      width="640" height="360"
      controls></video-js>
    <script src="https://players.brightcove.net/1752604059001/default_default/index.min.js"></script>
  </div>

        <!-- Script for the picture-in-picture plugin -->
        <script src="https://players.brightcove.net/videojs-pip/1/videojs-pip.min.js"></script>

      <div>上記の要素がプレーヤーの親として見つからない場合、プラグインは初期化に失敗し、次のログ警告が表示されます。

      expected player to be a child of a "vjs-pip-container" element, cannot continue with picture-in-picture plugin
    4. 次のことを行うカスタムスクリプトコードを追加します。
      • プレーヤーの準備ができたら、Brightcoveプレーヤーへの参照を取得します。この例では、myPlayerという名前の変数を作成し、プレーヤーへの参照を割り当てます。
      • ピクチャーインピクチャープラグインを初期化します。
      videojs.getPlayer('myPlayerID').ready(function() {
  // When the player is ready, get a reference to it
  var myPlayer = this;
  // Initialize the picture-in-picture plugin
  myPlayer.pip();
});

      詳細については、 codepenの例上記。

    オプション

    初期化時に options オブジェクトをプラグインに渡すことができます。このオブジェクトには、次のいずれかのオプションを含めることができます。

    allowOnMobile

    • allowOnMobile:
      • タイプ: boolean
      • デフォルト: false

      • デフォルトでは、ピクチャーインピクチャーモードはAndroidまたはiOSモバイルデバイスでは機能しません。これは、これらのCSS固定位置が、デスクトップデバイスの場合と同じように、複数のビューポートを持つズーム可能なデバイスでは機能しないためです。

        サポートされているモバイルデバイスでピクチャーインピクチャーを有効にする場合は、このオプションを使用できます。サポートを有効にしたい場合があります。

        • インテグレータは、ピクチャーインピクチャープレーヤーモードのポジショニングを管理する責任を負います。
        • プレーヤーを使用しているWebサイトがズームされる可能性はほとんどありません。
        • を使用してズームが無効になっているuser-scalable=no指令。このディレクティブを使用すると、position: fixedデスクトップと同じように動作しますが、アクセシビリティの問題になる可能性があるため、特にお勧めしません。 
          <meta name="viewport" content="width=device-width, user-scalable=no">

      • 例:

        // Allow mobile (iOS and Android) devices to enter PIP mode.
player.pip({allowOnMobile: true});
    バージョン12.2より前のiOSデバイスおよびAndroidデバイスでは、プレーヤーが新しい場所に移動すると、元のプレーヤーの場所がブラウザーに完全に表示されていても、元のプレーヤーの場所に戻ることはありません。

    クローズ可能

    • closeable:
      • タイプ: boolean
      • デフォルト: true
      • デフォルトでは、ユーザーは[ピクチャーインピクチャー]をクリックして、[ピクチャーインピクチャー]モードを無効にできます。バツプレーヤーの右上隅にあるボタン。この機能は、渡すことで無効にできますfalseこのオプションの場合。

      • 例:

        // Do not allow the user to close the PIP mode player.
player.pip({closeable: false});

    規模

    • scale:
      • タイプ: number
      • デフォルト: 2 / 3
      • プレーヤーがピクチャーインピクチャーモードのときにプレーヤーに適用される倍率。この値は、ゼロより大きく1以下の数値である必要があります。

      • 例:

        // Detach the player, but do not change its size.
player.pip({scale: 1});

      • 例2: 

        // Detach the player, and change its size to 1/2.
player.pip({scale: 0.5});

    高さと幅

    • height and width:
      • タイプ: number
      • デフォルト: null

      • デフォルトでは、プラグインはプレーヤーのサイズを、scaleオプション。ただし、height（またはwidth）はデフォルトのスケーリングを上書きし、スケールダウンされたプレーヤーのサイズを明示的に設定します。

        1つの寸法のみが指定されている場合、アスペクト比を維持するためにもう1つの寸法が縮小されます。両方の寸法が指定されている場合、プレーヤーは正確に指定されたサイズに設定され、アスペクト比を変更できます。

      • 例:

        // Detach the player and set its width to 300 pixels. Scale its height to
// maintain its current aspect ratio.
player.pip({width: 300});

    posX

    • posX:
      • タイプ: string
      • デフォルト: "right"

      • ピクチャーインピクチャーモードのときのプレーヤーの水平方向の配置-「右」または「左」。

      • 例:

        // When the player is in PIP mode, align it to the left side of the viewport.
player.pip({posX: 'left'});

    posY

    • posY:
      • タイプ: string
      • デフォルト: "bottom"

      • ピクチャーインピクチャーモードのときのプレーヤーの垂直方向の配置-「上」または「下」。

      • 例:

        // When the player is in PIP mode, align it to the top of the viewport.
player.pip({posY: 'top'});

    視認可能

    • viewable:
      • タイプ: number
      • デフォルト: 0.8

      • プレーヤーが表示可能と見なされるしきい値。つまり、プレーヤーの総面積（高さと幅）のこの割合がブラウザのビューポートに表示されている場合、表示可能と見なされます。

        たとえば、デフォルトの0.8では、プレーヤーの80％がビューポートに表示されない限り、プレーヤーは表示可能とは見なされません。ザ・viewable値は、0以上1以下の数値である必要があります。

      • 例:

        // If more than half the player is outside of the viewport, activate PIP mode.
player.pip({viewable: 0.5});

    manualContainerSize

    • manualContainerSize:
      • タイプ: boolean
      • デフォルト: false

      • デフォルトでは、このプラグインが有効になっているプレーヤーは、特別なコンテナ要素の物理的な寸法をプレーヤーの寸法と同期させます。ただし、この動作はすべてのユースケースで機能するとは限らないため、このオプションをに設定することで無効にできます。true。

        その場合、コンテナ要素は通常のブロック要素のように動作します。これは、プラグインのユーザーが自分でそのサイズを管理する必要があることを意味します。

        このオプションは、ブール値を介して埋め込みコードで設定することもできますdata-manual-container-sizeコンテナの属性。その存在により、このオプションがtrueに設定されます。プラグインに渡される値は、埋め込みコードで定義されている値よりも優先されることに注意してください。

      • 例:

        // Implementation will handle sizing the container.
player.pip({manualContainerSize: true});

      • を使用した例data-manual-container-sizeコンテナの属性：

        <div class="vjs-pip-container" data-manual-container-size>
  <video-js class="video-js vjs-default-skin">
  </video-js>
</div>

    オプションの使用

    このオプションを利用するには、次の 2 つの方法があります。

    1. スタジオの「 > プレーヤーのプラグイン」セクションにあります。
    2. プレーヤーでJavaScriptを使用します。

    スタジオを使用する

    Studioで、プレーヤーを編集して、に戻ります。プラグインセクション。名前リストのPicture-in-Pictureをクリックします。適切なJSON形式を使用して、オプションを（引用符で囲んで）リストし、その後に適切な値をリストします。値が文字列の場合は、引用符で囲む必要があります。数値またはブール値の場合、引用符を含めることはできません。

    JavaScriptを使用する

    コードでオプションを実装するには、オブジェクトを作成し、必要なオプションをそれぞれの値を割り当てて、プラグインを呼び出すときにオプションオブジェクトを渡します。

    videojs.getPlayer('myPlayerID').ready(function() {
  // When the player is ready, get a reference to it
  var myPlayer = this,
      options = {};

  options.scale = 0.5;
  options.posX = "left";
  // Initialize the picture-in-picture plugin
  myPlayer.pip(options);
});

    メソッド

    次のメソッドを使用すると、ピクチャーインピクチャープラグインを操作できます。

    方法 説明
    myPlayer.pip().enable() 自動ピクチャーインピクチャー動作を有効にする
    myPlayer.pip().disable() 自動ピクチャーインピクチャー動作を無効にする
    myPlayer.pip().toggle() 現在の状態に基づいて、ピクチャーインピクチャーモードを手動でアクティブまたは非アクティブにします
    myPlayer.pip().activate() プレーヤーを手動でピクチャーインピクチャーモードにする
    myPlayer.pip().deactivate() プレーヤーを手動でピクチャーインピクチャーモードから解除します

    イベント

    次のイベントは、picture-in-pictureプラグインから発生します。

    イベント 説明
    beforepipactive ピップモードがアクティブになる前に発火します
    pipactive ピップモードがアクティブになった後に発生します
    beforepipinactive ピップモードが無効になる前に発火します
    pipinactive ピップモードが無効になった後に発生します
    beforepipenabled videojs-pipプラグインが有効になる前に起動します
    pipenabled videojs-pipプラグインが有効になった後に発生します
    beforepipdisabled videojs-pipプラグインが無効になる前に起動します
    pipdisabled videojs-pipプラグインが無効になった後に発生します
