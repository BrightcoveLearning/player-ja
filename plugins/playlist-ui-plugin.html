    プレイリスト UI プラグイン

    このトピックでは、プレイリストUIプラグインを使用して、デフォルトのプレイリスト機能のルックアンドフィールを強化する方法を学習します。

    概要

    プレイリストUIプラグインには、プレイリストの動作をカスタマイズするために使用できるオプションが含まれています。多数のオプションにより、レイアウト、動作、実装戦略など、プレイリストに変更が加えられます。次の例は、プレイリストの2つの基本的なレイアウト（垂直方向と水平方向）を示しています。例ごとに、カスタマイズできるものを紹介するための動作の小さなセットがリストされています。

    次のドキュメントは、プレイリストの使用方法を完了するための重要な情報です。
    • プレイリストの実装 :このドキュメントでは、プレイリストの使用に関して行う必要がある決定と、HTML ページで実際にプレイリストを取得するために必要なプレーヤーコードを取得する方法について説明します。
    • プレイリストのスタイリング :このマニュアルでは、プレイリストの外観をカスタマイズするための情報を提供します。

    縦型レイアウト

    以下のプレイリストの例は、次のとおりです。

    • プレーヤーの右側にある垂直プレイリストには表示/非表示ボタンがあり、プレイリストは自動的にサイズ変更および配置されます。これは、を使用する場合のデフォルトの動作です。標準（iframe）プレーヤーの実装。
    • プレイリストが最初に表示されます。これは、を使用して変更できますhideOnStartオプション。
    • ビデオの終わり近くに、次のビデオと次のビデオのサムネイルまでの時間を表示するオーバーレイが表示されます。これはデフォルトの動作であり、nextOverlayオプション。

    水平レイアウト

    以下のプレイリストの例は、次のとおりです。

    • 動画の下にある横向きの再生リスト。
    • 上記の例のように、オーバーレイがビデオの終わり近くに表示されるのと同様に、エンドスクリーンには次のビデオの再生が開始されるまでの時間が表示されます。これは、autoadvanceオプションがゼロ以外の場合、ビデオ間の一時停止の終了画面が表示されます。エンドスクリーンは、を使用して変更できますnextEndscreenオプション。

    プレーヤー/プレイリストの関連付け

    デフォルトでは、プレイリストUIプラグインは、特定のプレーヤーの正しいプレイリストコンテナ要素の検索を処理します。これは、最初の空が見つかることを意味します.vjs-playlist DOMの要素とそれを使用します。ただし、プラグインは、複数のプレーヤーで複雑なワークフローを構築している場合に、プレーヤーとプレイリストコンテナの間で作成できるより明示的な関連付けを提供します。

    data-for attribute

    ザ・data-for属性をプレイリストコンテナに適用して、プレーヤーのコンテナに関連付けることができますid、 例えば： 

    <video-js id="myPlayerID"
  ...></video-js>

<div class="vjs-playlist" data-for="myPlayerID"></div>

    これは、利用可能な明示的な関連付けの最も具体的な方法です。他の方法よりも優先されます。

    data-player and data-embed attributes

    ザ・data-playerそしてdata-embed属性をプレイリストコンテナに適用して、Brightcoveプレーヤーに関連付けることができます。アソシエートが正しく機能するには、両方の属性を使用する必要があることに注意してください。次の例では、2番目のみ<div>タグは、特定の関連付けが行われるときにプレーヤーのプレイリストを保持します。最初<div>空になります。 

    <video-js data-playlist-id="5455901760001"
	data-account="1507807800001"
	data-player="SJLNAJye7"
	data-embed="default"...></video-js>
<script src="https://players.brightcove.net/1507807800001/SJLNAJye7_default/index.min.js"></script>

<div class="vjs-playlist"></div>

<div class="vjs-playlist" data-player="SJLNAJye7" data-embed="default"></div>

    オプション

    初期化時に options オブジェクトをプラグインに渡すことができます。このオブジェクトには、次のいずれかのオプションを含めることができます。

    次のセクションでは、コードとStudioの両方を使用してオプションを使用する方法について詳しく説明します。

    オートアドバンス

    • autoadvance:
      • タイプ: number
      • デフォルト: undefined
      • 再生リスト内の動画の間に一時停止が発生するかどうか、またその期間を決定します。詳細については、オートアドバンスセクションプレイリストAPIガイド。

    hideOnStart

    • hideOnStart:
      • タイプ: boolean
      • デフォルト: false
      • プレイリストを最初に非表示にするかどうかを決定します。これは、を使用する場合にのみ機能しますiframeプレイリストの表示/非表示機能はiframeでのみ使用できるため、プレーヤーの実装は論理的です。このオプションは、水平プレイリストでは機能しません。
      • 要件/依存関係:
        • playlistPicker: true
        • iframe埋め込み
        • horizontal: false

    水平

    • horizontal:
      • タイプ: boolean
      • デフォルト: false
      • プレイリストを垂直方向ではなく、プレーヤーの下に水平方向に表示します。
      • 要件/依存関係:
        • playlistPicker: true

    nextButton

    • nextButton:
      • タイプ: boolean
      • デフォルト: true
      • 場合true、次のプレイリストアイテムに移動するためのボタンが追加されます。オプションをに設定すると、ボタンを無効にできますfalse。ボタンは、再生ボタンの右側のコントロールバーに追加されます。

    nextEndscreen

    • nextEndscreen:
    • タイプ: boolean
    • デフォルト: true
    • 場合true、再生後、プレーヤーはプレーヤーの上にモーダルエンドスクリーンを表示します。オプションをに設定すると、エンドスクリーンを無効にできます。false。このエンドスクリーンは、再生リスト内の次のビデオをプレビューします。オプションに関するモードの詳細は次のとおりです。
      • ザ・autoadvanceオプションはゼロより大きい値に設定する必要があります（そうしないと、終了画面を表示する時間がなく、スキップされます）。
      • 表示されるカウントダウンは、autoadvance引き金。
      • プレイヤーはしなければならないないビデオの後に表示するように構成されたカスタムエンドスクリーンプラグインまたはソーシャルプラグインの実装を含めます。いずれかの状態が検出された場合、終了画面は表示されません。

    nextOverlay

    • nextOverlay:
      • タイプ: boolean
      • デフォルト: true
      • 場合true、プレーヤーは、プレーヤーの右下隅に小さなオーバーレイを表示します。オプションをに設定すると、オーバーレイを無効にできます。false。このオーバーレイは、再生リスト内の次のビデオをプレビューします。オプションに関するモードの詳細は次のとおりです。
        • 後続のプレイリストアイテムが必要です。
        • ザ・autoadvanceオプションはゼロ以上に設定する必要があります。
        • カウントダウンは、動画の残り時間と任意の時間を表しますautoadvance待ち時間。
        • オーバーレイは、ビデオの終わりから10秒で表示されます。ただし、動画の長さが30秒未満の場合は、再生時間の2/3ポイントに表示されます。
        • オーバーレイがユーザーによって閉じられた場合、ソースが変更されるまで、オーバーレイは閉じられたままになります。新しいソースで再表示されます。
        • の場合次のエンドスクリーンまでを有効にすると、ビデオが終了するとオーバーレイが非表示になります。それ以外の場合は、次のビデオまで開いたままになります。

    プレイリスト

    • playlist:
      • タイプ: array
      • デフォルト: undefined
      • 初期プレイリストデータを渡すために使用されます。詳細については、プレイリストセクションプレイリストAPIガイド。

    プレイリストピッカー

    • playlistPicker:
      • タイプ: boolean
      • デフォルト: true
      • ユーザーが動画をクリックできるように、再生リストに動画の視覚的なリストを含めるかどうかを決定します。ザ・hideOnStartそしてhorizontalオプションは、その外観と動作を変更します。いつplaylistPickerですfalse、他のUI要素は引き続きnextButton、nextEndscreenそしてnextOverlayオプション。

    PlayonSelect

    • playOnSelect:
      • タイプ: boolean
      • デフォルト: false
      • ザ・playOnSelectオプションは、ユーザーが再生リスト内のビデオをクリックしたときにビデオの再生を開始するかどうかを制御するために使用されます。オプションがに設定されている場合true、再生リストから新しい動画を選択すると、プレーヤーが以前に一時停止されていた場合でも、その動画の再生が開始されます。デフォルトでは、プレイリストから新しい動画をクリックすると、新しい動画が読み込まれますが、あらかじめ一時停止していた場合は、プレーヤーは一時停止したままになります。
      • 要件/依存関係:
        • playlistPicker: true

    繰り返す

    • repeat:
      • タイプ: boolean
      • デフォルト: false
      • 再生リストの最後の動画が終了した後、再生リストを繰り返します。この機能は、プレイリストの最後のビデオが終了すると、プレイリストの最初のビデオを再生します。

    showDescription

    • showDescription:
      • タイプ: boolean
      • デフォルト: false
      • 各プレイリストアイテムのビデオの説明を表示します。

    シャッフル

    • shuffle:
      • タイプ: boolean
      • デフォルト: false
      • 新しいデータが読み込まれるたびに、プレイリストアイテムをシャッフルします。

    オプションの使用

    このオプションを利用するには、次の 2 つの方法があります。

    1. スタジオの「 > プレーヤーのプラグイン」セクションにあります。
