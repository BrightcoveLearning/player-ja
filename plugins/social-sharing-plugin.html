    ソーシャル共有プラグイン

    このトピックでは、ソーシャルメディアプラグインを使用して、ユーザーがFacebook、Google +、Twitter、Tumblr、Pinterest、LinkedInを介してビデオを共有できるようにする方法を学習します。

    例

    BrightcovePlayerが掲載されたWebページがあるとします。ここで、ユーザーがソーシャルメディアプラットフォーム間でビデオを共有できるようにします。ソーシャルメディアプラグインを使用すると、タイトルと説明のドックにソーシャル共有アイコンが表示されます。

    注：タイトルと説明のドックを表示しないようにプレーヤーが構成されている場合、共有アイコンがプレーヤーのコントロール領域に表示されます。

    このアイコンをクリックすると、次に示すようなソーシャル共有画面が表示されます。

    次の2つの理由から、プレーヤーの幅と高さを設定する必要があります。

    1. 幅または高さが設定されていない場合、Twitterに渡す値がないため、Twitterカードが検証されません。
    2. 上のスクリーンショットに示すように、幅はソーシャルプラグインのインターフェースに対応するのに十分な幅である必要があります。インターフェースを使用して、ビデオ側にインターフェースを自動的に表示することもできます。displayAfterVideo プロパティ、以下に記載されています。

    注：Studioには、プレーヤーにソーシャルシェアリングを追加するオプションがあり、多くの場合、ニーズに十分対応できます。このドキュメントでは、ソーシャルメディアプラグインの使用に関するすべての側面について説明します。次のスクリーンショットは、StudioのUIを示しています。

    クリックここにプラグインを使用し、ソーシャルメディアのメタタグを含むWebページの実用的なサンプル。

    概要

    このドキュメントでは、BrightcovePlayerを使用して既存のHTMLページのソーシャルメディア共有を有効にする手順について説明します。これは、適切な構成情報を使用してBrightcoveプレーヤーを作成し、そのプレーヤーをHTMLページに追加したことを前提としています。

    ソーシャルメディアプラグインを追加するには、次の手順に従います。

    1. プレーヤーを構成します。これにより、ソーシャルメディアプラグインを使用するようにBrightcoveプレーヤーが更新されます。プレーヤーとのソーシャル共有を含めるには、次の3つの方法があります。

      • プレーヤーのプロパティを構成する
        スタジオでプレイヤーモジュール、あなたはすることができますでプロパティを構成しますソーシャルセクション。これは、さまざまなソーシャルメディアプラットフォームを選択するためのインターフェイスを提供しますが、他の構成オプションは提供しません。

      • オプションでプレーヤーを構成する
        スタジオでプレイヤーを使用するモジュールプラグインセクションには、構成オプションを提供するためのインターフェースがあります。

      • 共有URLのカスタマイズ
        この実装は、コードを使用してソーシャルプラグインを追加します。

    2. ソーシャルを追加メタタグプレーヤーのHTMLページに移動します。
    3. 検証 FaceBookおよびTwitterのHTMLページのメタタグ。

    ブラウザで、BrightcoveプレーヤーでHTMLページを開き、プレーヤーのソーシャル共有アイコンを使用してビデオを共有します。

    ページコード内にソーシャルメディアプラグインを実装することもできますが、これはより高度なソリューションです。このアプローチを使用して、カスタム共有URLを含めることができます。詳細については、共有URLのカスタマイズセクション。

    ソーシャルプラットフォームに投稿するための他のオプションは次のとおりです。

    • Facebook InstantArticlesでBrightcoveプレーヤーを公開する
    • ソーシャルメディアプラットフォームへのビデオの公開

    プレーヤーを構成する

    ソーシャル共有プラグインを追加するには、次の手順に従います。

    1. PLAYERS モジュールを開き、新しいプレーヤーを作成するか、プラグインを追加するプレーヤーを見つけます。
    2. プレーヤーのリンクを選択して、プレイヤーのプロパティを開きます。
    3. 選択するソーシャル左側のナビゲーションメニューにあります。
    4. 次に、のチェックボックスを選択します共有を有効にする。
    5. ビデオを共有したいソーシャルメディアプラットフォームを選択します。
    6. プレーヤーを公開するには、[ パブリッシュと埋め込み] [> 変更を公開 ] を選択します。
    7. 開くダイアログを閉じるには、[ 閉じる ] を選択します。

    オプションでプレーヤーを構成する

    より高度な構成については、プロパティプラグインを構成するためのオプションとして。これを行うには、次の手順を実行します。

    1. PLAYERS モジュールを開き、新しいプレーヤーを作成するか、プラグインを追加するプレーヤーを見つけます。
    2. プレーヤーのリンクを選択して、プレイヤーのプロパティを開きます。
    3. 左側のナビゲーションメニューで [ Social ] を選択し、チェックボックスがオフになっていることを確認します。
    4. 左側のナビゲーションメニューで [ プラグイン ] を選択します。
    5. 次に選択プラグイン 3つのオプションから。
    6. からプラグインを追加するドロップダウンメニューで、カスタムプラグイン。
    7. [ プラグイン名 ] に、と入力しますsocial。
    8. JavaScript の URL には、次のように入力します。 
      		https://players.brightcove.net/videojs-social/3/videojs-social.min.js
    9. CSS の URL には、次のように入力します。 
      		https://players.brightcove.net/videojs-social/3/videojs-social.css
    10. [ オプション (JSON) ] テキストボックスに構成オプションを入力します。この例では、オプションは次のことを行います。
      • 2-3行目は次のとおりです。動画の名前と説明を設定します。
      • 4行目：ビデオプレーヤーとソーシャル共有メタタグを使用して、カスタムWebページのURLを定義します。
      • 5〜6行目：Twitterの場合、これにより、ユーザーは5秒のオフセット後にビデオの表示を開始できます。

      • 7〜13行目：Facebook、Twitter、Tumblr、Pinterest、LinkedInのソーシャル共有を可能にします。

      	{
		"title": "Tiger",
		"description": "Video of two tigers",
		"url": "https://solutions.brightcove.com/bcls/brightcove-player/social-plugin/social-media.html",
		"deeplinking": true,
		"offset": "00:00:05",
		"services": {
			"facebook": true,
			"twitter": true,
			"tumblr": true,
			"pinterest": true,
			"linkedin": true
		}
	}
    11. [保存]をクリックします。
    12. プレーヤーを公開するには、[ パブリッシュと埋め込み] [> 変更を公開 ] をクリックします。
    13. 開くダイアログを閉じるには、[ 閉じる ] をクリックします。

    プロパティ

    一般的なプラグインの場合、options オブジェクトは、初期化をカスタマイズするためにプラグインにデータを渡すために使用されます。この場合、このオブジェクトで次のプロパティを使用できます。

    タイトル

    • 型:文字列
    • これは、動画が共有されたときに表示されるカスタムタイトルです。

    説明

    • 型:文字列
    • これは、それをサポートするソーシャルサービスによって使用されるカスタムの説明です。

    URL

    • 型:文字列
    • これは、共有用のビデオとメタタグを含むカスタムWebページを指すURLです。を参照してくださいメタタグソーシャルメディアメタデータをプレーヤーページに追加する方法の詳細については、セクションを参照してください。

    埋め込みコード

    • 型:文字列
    • これは、ビデオを共有するためのBrightcove Playeriframe埋め込みコードです。これにより、の内容を完全に上書きできます埋め込みコード共有ダイアログにあるフィールド。
    • このプロパティの値は、 getEmbedCode（）方法。

    removeEmbed

    • 型:ブール値
    • デフォルト: false
    • に設定されている場合true、iframe埋め込みコードをオフ（非表示）にします

    displayAfterVideo

    • 型:ブール値
    • デフォルト: false
    • ソーシャル画面を自動的に表示しますended イベント

    ディープリンク

    • 型:ブール値
    • デフォルト: false

    • これにより、共有ビデオのディープリンクが有効/無効になります。この機能により、ユーザーは特定のオフセットからビデオの視聴を開始できます。詳細については、ディープリンク資料。

      注：ディープリンク機能は現在、Twitterでのみ機能します。

    offset

    • 型:文字列
    • フォーマット：00h00m00s
    • これは、deeplinking プロパティであり、ビデオの再生を開始するタイミングを定義します。1分5秒をオフセットするには、次のように記述します。00h01m05s

    サービス

    このオブジェクトにすべてのサービスプロパティを含めます。サービスのサポートを有効または無効にするには、プロパティ値を削除するのではなく、trueまたはfalseに設定します。

    • facebook
      • 型:ブール値
      • デフォルト: true
      • これにより、Facebook共有アイコンが有効になります。
    • twitter
      • 型:ブール値
      • デフォルト: true
      • これにより、Twitter共有アイコンが有効になります。
    • tumblr
      • 型:ブール値
      • デフォルト: true
      • これにより、Tumblr共有アイコンが有効になります。
    • pinterest
      • 型:ブール値
      • デフォルト: true
      • これにより、Pinterest共有アイコンが有効になります。
    • linkedin
      • 型:ブール値
      • デフォルト: true
      • これにより、LinkedInの共有アイコンが有効になります。

    検証

    手順に従ってページにメタタグを含めると、FacebookとTwitterの両方でページを検証できます。いずれかの検証プロセスでエラーが発生した場合は、必要に応じてプレーヤーの構成を更新してください。

    Facebook

    Facebookの場合は、 Facebookデバッガーメタタグを含むページにURLを貼り付けて、デバッグ検証します。

    Twitter

    Twitterの場合は、カードバリデーター、メタタグを含むページにURLを貼り付けて、プレビューカード検証します。検証の追加の利点は、TwitterがWebサイトの検証を要求し、待機リストに登録されてTwitterカードが承認されることです。

    共有URLのカスタマイズ

    ソーシャルメディアプラグインのより高度な実装は、ページ内埋め込みコード Playersモジュールから。これにより、ページコード内から構成オプションを渡すことができます。これを行う理由の1つは、カスタムを提供することですembedCode ページ上のプレーヤーとは異なるプレーヤーを共有するための値。

    カスタムコードを使用してプラグインを実装するには、次の手順に従います。

    1. PLAYERS モジュールを開き、新しいプレーヤーを作成するか、プラグインを追加するプレーヤーを見つけます。
    2. プレーヤーのリンクを選択して、プレイヤーのプロパティを開きます。
    3. 左側のナビゲーションメニューで [ Social ] を選択し、チェックボックスがオフになっていることを確認します。
    4. 左側のナビゲーションメニューで [ プラグイン ] を選択します。
    5. 選択するスタイルシートを選択しますスタイルシートを追加するボタン。
    6. CSS の URL には、次のように入力します。 
      		https://players.brightcove.net/videojs-social/3/videojs-social.css
    7. 選択するスクリプトを選択しますスクリプトを追加するボタン。
    8. JavaScript の URL には、次のように入力します。 
      		https://players.brightcove.net/videojs-social/3/videojs-social.min.js

    カスタムサービス

    プラグインには、6つの人気のソーシャルメディアプラットフォームのサポートが組み込まれていますが、カスタムサービスを使用して他のプラットフォームを追加することもできます。これを行うには、options.customServices オブジェクト。このオブジェクトは、カスタムサービスオブジェクトの配列です。構成の簡単な例を示し、次に例の詳細を示します。

    次の例では、3つのボタンを追加します。XingとWhatsAppの最初と2番目のボタン。WhatsAppボタンはモバイルでのみ表示されます。これらの2つの例では、アイコンにFont Awesomeを使用しています。これは、プレーヤーがすでにフォントを使用しているページにページ内に埋め込まれている場合に便利ですが、他のスタイルを使用することもできます。3つ目は、FacebookのフィードAPIを使用したFacebook共有ボタンです。これにより、FacebookアプリIDを帰属に含めることができます。これを使用するときは、通常のFacebookボタンを無効にすることをお勧めします。

    	[{
		className: 'fa fa-xing',
		title: 'Xing',
		hrefTemplate: 'https://www.xing.com/spi/shares/new?url=',
		bgColor: '#026466'
	},
	{
		className: 'fa fa-whatsapp',
		title: 'WhatsApp',
		hrefTemplate: 'https://api.whatsapp.com/send?phone=&text=title%20',
		bgColor: '#25d366',
		mobileOnly: true
	},
	{
		classSuffix: 'facebook',
		title: 'Facebook',
		hrefTemplate: 'https://www.facebook.com/dialog/feed?app_id=123456789&caption=&display=page&link='
	}]

    以下は、カスタムサービスオブジェクトのプロパティを定義します。

    customService.title

    • 型:文字列
    • 共有先のタイトル。

    customService.className

    • 型:文字列
    • ボタンに追加するクラスまたはスペースで区切られたクラスのリスト。

    customService.classSuffix

    • 型:文字列
    • 接尾辞はに追加されますvjs-icon- ボタンのクラスを生成します。次の場合にのみ使用されますclassName 指定されていません。Video.jsフォントには追加のソーシャル宛先アイコンが含まれていないため、標準の宛先の1つを置き換える場合にのみ役立ちます。

    customService.hrefTemplate

    • 型:文字列
    • マクロを含む可能性のある共有URLを生成するためのstingテンプレート 、 、 そして 。代わりに使用できますcustomService.href 関数を指定できない場合。たとえば、JSON文字列でオプションをPlayer ManagementAPIに提供します。

    customService.href

    • 型:文字列
    • 共有URLを生成する関数。オブジェクトを取得して文字列を返す必要があります。特に指定がない限り、これはから生成されますhrefTemplate 。

    customService.bgColor

    • 型:文字列
    • 背景色要素スタイルとして適用されるオプションのCSS色値。ボタンがCSSでスタイル設定されている場合は必須ではありませんが、特にアイコンが既存のスタイルシートでスタイル設定される場合に便利です。

    customService.mobileOnly

    • 型:ブール値
    • デフォルト:false
    • 場合true 、ボタンはAndroidとiOSでのみ表示されます。

    既知の問題

    エンドスクリーンに簡単な説明はありません

    ビデオの簡単な説明動画の再生中に共有ボタンをクリックするとが表示されます。共有情報を含むエンドスクリーンでは、簡単な説明は表示されません。これは設計どおりです。

    ボタンのキーボードナビゲーション-Firefoxfor OS X

    Firefox for OSXには、テキストボックスをタブで移動し、要素のみを一覧表示するデフォルトのタブ動作があります。ソーシャルプラグインのボタンはリンク要素であるため、チェックボックスをオンにしない限り、キーボードを使用してボタンをナビゲートすることはできません。すべてのコントロール下システム環境設定>キーボード>ショートカット。
