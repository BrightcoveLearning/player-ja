    Brightcove ブプレーヤー用 Tealium プラグイン

    このトピックでは、BrightcovePlayerでTealiumiQタグマネージャーを使用する方法を学習します。

    概要

    Tealiumプラグインを使用すると、TealiumiQタグマネージャーをBrightcovePlayerと統合できます。これにより、Tealium Universal DataHubを使用してBrightcovePlayerイベントを追跡できます。

    タグマネージャを使用すると、コードを使用せずにタグを追加および公開できます。タグマネージャは、クライアント側のソリューションであるため、さまざまなマーケティング技術にわたってデータを収集できます。これにより、マーケティング担当者は、多様な収集ポイントにわたってユーザーとプレーヤーのデータを収集できるようになります。

    Tealium iQは特定のプラットフォーム向けに設計されていないため、お好みの製品で使用できます。Tealium iQを使用すると、ほとんどのニーズに対応するタグテンプレートを見つけることができるタグマーケットプレイスにアクセスできます。

    Tealium Tag Managerプラグインは、BrightcovePlayerバージョン6.36.5以降でサポートされています。

    タグ管理の用語

    TealiumiQを使用するときに遭遇する一般的な用語のいくつかを次に示します。

    用語 説明
    イベント イベントは、レポートツールにデータを送信するために使用されるデータポイントです。これらは、ビデオの読み込みと再生に関連しています。
    プロフィール タグ管理システムアカウントは、独自のタグと変数を持つ複数のコンテナ (プロファイル) を持つことができます。各プロファイルには複数の環境を含めることができ、通常は開発、ステージング、および本番用にセットアップされます。
    ルール ルールは、タグコードをページに追加するタイミングを指示します。これらはユーザー定義の条件です。
    タグ タグは、ユーザーインタラクションデータを収集するために Web ページに配置されるコードスニペットです。タグ管理システムを使用すると、サイトにタグを追加するプロセスが簡素化され、タグ ID とタグイベントタイプのみを指定する必要があります。
    タグマーケットプレイス タグマーケットプレイスのテンプレートは、タグを展開するためのコードスニペットを提供します。これにより、必要なコード知識の量が減りますが、実装を成功させるには JavaScript と HTML に精通している必要があります。
    トラッカー トラッカーは、データを収集、保存、Google タグマネージャに送信するオブジェクトです。
    ユニバーサルデータオブジェクト（UDO）utag_dataオブジェクト と呼ばれるJavaScript変数utag_dataこれには、Webページのコードに存在するデータのコレクションが含まれています。この JavaScript オブジェクトは、Web サイトからタグマネージャコンテナに情報を渡すために使用されます。次に、この情報を使用して変数を設定し、タグ構成でイベントをアクティブ化できます。

    入門編

    を確認しますTealiumiQタグ管理アカウントを設定するためのドキュメント。セットアップには次のものが含まれます。

    • タグ構成を定義する
    • ビーコンを発射するトリガーを定義する

    • イベントとカスタムディメンションの変数を作成します

    Brightcoveプラグインの使用

    他のBrightcovePlayerプラグインと同様に、Tealiumプラグインをプレーヤーにロードする必要があります。

    スタジオを使用する

    以下の手順では、Video Cloud Studio を使用してプラグインをロードする方法について詳しく説明します。

    1. PLAYERS モジュールを開き、新しいプレーヤーを作成するか、プラグインを追加するプレーヤーを見つけます。
    2. プレーヤーのリンクを選択して、プレイヤーのプロパティを開きます。
    3. 左側のナビゲーションメニューで [ プラグイン ] を選択します。

    4. 次に、[ プラグインを追加 ] ボタンを選択し、[ Brightcove プラグイン ] を選択します。

    5. を展開しますBrightcoveプラグインドロップダウンして選択しますTealium。

    6. これで、追跡するディメンションを使用してGoogleAnalyticsを構成する準備が整いました。利用可能な寸法を確認してください。イベントと属性このドキュメントのセクション。

    7. の中にオプション（JSON）テキストボックスに、追跡するイベントと属性を入力します。フィールドの説明:

      • paramsToTrack - List of attributes you want to track
      • eventsToTrack - List of events you want to track
      • account - Tealium account name
      • profile - Tealium profile name
      • env - Tealium environment name
       

      例を次に示します。

      {
  "paramsToTrack": {
    "video_playhead": "video_playhead",
    "video_platform": "video_platform",
    "video_length": "video_length",
    "video_name": "video_name",
    "video_id": "video_id",
    "video_milestone": "video_milestone"
  },
  "eventsToTrack": {
    "video_load": "video_load",
    "video_play": "video_play",
    "video_complete": "video_complete",
    "video_watch": "video_watch",
    "video_pause": "video_pause",
    "ad_start": "ad_start",
    "ad_end": "ad_end"
  },
  "env": "your environment",
  "profile": "your profile",
  "account": "your account"
}

    8. [ 保存 ] ボタンを選択します。これで、 Tealiumプレーヤーのプラグインのリストにプラグインが追加されました。

    9. プレーヤーを公開するには、[ パブリッシュと埋め込み] [> 変更を公開 ] を選択します。
    10. ダイアログを閉じるには、[ 閉じる ] を選択します。

    11. に戻るメディア Tealium用に更新したプレーヤーを使用してビデオまたはプレイリストを公開します。

      プレーヤーがロードされ、ユーザーがプレーヤーと対話すると、指定されたイベントとパラメータが追跡されます。

    イベントと属性

    以下のイベントと属性をTealiumiQに送信できます。

    イベント

    これらのイベントは、あなたのプレーヤーから来ます。これらのイベントはすべて、共通の属性を送信します。一部のイベントは、追加の属性を送信する場合があります。

    イベント タイプ 説明
    video_load 標準 ビデオメタデータのプレーヤーへの読み込みが完了したとき。
    video_play 標準 ユーザーまたは自動再生のいずれかによって再生が開始された場合
    送信される追加の属性：
    • video_playhead
    video_watch [1] カスタム これには、追加のレンディションの選択、ビットレート測定、およびバッファリング情報が含まれます。
    Additional attributes sent:
    • video_playhead
    • video_milestone
    video_pause 標準 ユーザーがビデオの再生を一時停止したとき
    送信される追加の属性：
    • video_playhead
    ad_start カスタム 広告の再生が開始されたとき
    ad_end カスタム 広告の再生が終了したとき
    video_complete 標準 ビデオの再生が終了するとトリガーされます。
    注

    以下の注記は、上記の表を参照しています。

    [1]ザ・video_watchイベントは、次のマイルストーンで送信されます。

    • 25％のマイルストーン–ビデオが総ビデオ時間の25％から50％の範囲で視聴された場合
    • 50％のマイルストーン-ビデオが総ビデオ時間の50％から75％の範囲で視聴された場合
    • 75％のマイルストーン-ビデオが総ビデオ時間の75％から95％の範囲で視聴された場合
    • 100% マイルストーン-動画の合計再生時間の 100% で動画が視聴されたとき

    ビデオ視聴率（video_watch）は次のように計算できます（GoogleおよびAdobeアナリティクスのvideo_engagementイベントと同様）。

    Video watched percentage = duration the user actually watched / total video duration

    例えば、20秒のビデオでは、以下の両方のケースで25％のマイルストーンを達成することができます

    • ユーザーは0～5秒から監視します
    • ユーザーは0-1秒から見て11秒にスキップして15秒に観る

    上記のいずれの場合も、ユーザーは 5 秒を視聴したため、マイルストーンは 25% として報告されます。

    共通の属性

    これらの属性は、すべてのイベントに共通です。

    名前 タイプ 説明
    video_id 文字列 イベントを開始したBrightcoveビデオID
    video_length 数値 ビデオ属性で指定されたビデオの長さ
    video_name 文字列 video属性で指定されたビデオの名前
    video_platform 文字列 ビデオが実行されているプラットフォーム

    イベント固有の属性

    これらの属性は、特定のイベントに関連しています。

    名前 タイプ 説明
    video_playhead 数値 イベントがトリガーされるビデオ内の時間。
    Sent with events:
    • video_play
    • video_pause
    • video_watch
    video_milestone 数値 イベントのトリガー時に完了したビデオの割合。
    Sent with event:
    • video_watch

    Tealiumでのデータの表示

    プレーヤーのアクションによりプラグインがトリガーされ、分析データがTealium Universal DataHubに送信されます。イベントと属性を表示するには、次の手順に従います。

    1. に移動https://my.tealiumiq.com
    2. 資格情報を使用してUniversalDataHubにログインします。
    3. 左側のナビゲーションで、展開しますEventStream。

    4. 左側のナビゲーションで、ライブイベント。

    5. リアルタイムのイベントデータを含むダッシュボードが表示されます。

    6. イベントを選択すると、そのイベントに関連付けられている属性が表示されます。
