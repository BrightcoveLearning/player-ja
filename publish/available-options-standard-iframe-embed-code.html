標準（iframe）埋め込みコードで使用可能なオプション

{{ page.description }}

はじめに

iframe プレーヤー実装コードは、プレーヤーを単一の包含ユニットとして使いやすさを提供します。これは、ソーシャルメディアのアプリケーションや共有にしばしば必要です。

使用法

注:Brightcove Playerをご利用の場合は、videoIdプレーヤーにビデオを割り当てるためのクエリパラメータ。プレーヤーモジュールでビデオを割り当てるか、curl を使用します。

を使用するにはiframeコードは次の手順に従います。

  1.  プレイヤーを作成するには、PLAYERS モジュールを使用します。
  2. MEDIA モジュールに移動し、新しく作成したプレーヤーを使用してビデオを公開します。
  3.  標準埋め込みコードをコピーします。HTML は、次のようになります。 
          <iframe src="https://players.brightcove.net/921483702001/default_default/index.html?videoId=4570583963001"
        allowfullscreen
        webkitallowfullscreen
        mozallowfullscreen></iframe>
  4. ブラウザから HTML をコピーし、完全な HTML ページのに貼り付けます。body
  5. HTML ページを参照して、プレーヤーが機能しているかどうかを確認します。

プリロードの詳細

preload パラメータは、ビデオタグがロードされるとすぐにビデオデータのダウンロードを開始する必要があるかどうかをブラウザに通知します。オプションは次のとおりです自動、メタデータ、および無し。HLS 再生技術の場合、設定は少し異なることを意味し、以下に記します。

  • 自動 (デフォルト):すぐにビデオの読み込みを開始します（ブラウザが同意した場合）。iPhoneやiPadなどの一部のモバイルデバイスでは、' ユーザーの帯域幅を保護するためにビデオをプリロードしません。これが、値が自動と呼ばれ、真のような最終的なものではない理由です。

    HLS再生テクノロジーの場合、自動これは、Brightcove Playerのバージョン5.3より前の場合のように、プレーヤーがm3u8マニフェストファイルと3〜4ではなく1つのセグメントのみをダウンロードすることを意味します。

  • メタデータ：ビデオのメタデータのみをロードします。これには、ビデオの長さやサイズなどの情報が含まれています。

    HLS再生テクノロジーの場合、メタデータと同じです自動。

  • なし :ビデオデータをプリロードしないでください。これは、ユーザーが [再生] をクリックしてダウンロードを開始するまで待機します。iOS では、この設定でもセグメントのダウンロードが発生する可能性があることに注意してください。詳細については、Brightcove Playerと iOSのドキュメントを参照してください。

    HLS再生テクノロジーの場合、なしつまり、ユーザーが再生を押すまで、プレーヤーはm3u8マニフェストを含む何もロードしなくなります。
      <video-js preload ...>
      or
      { "preload": "auto" }
注:サーバーサイド広告挿入 (SSAI) を使用する場合、ブライトコーブは設定することをお勧めしますpreload=none。もしもpreloadに設定されていますautoまたmetadata、ビデオストリームの最初のフレームが要求されます。これにより、ユーザーが再生を開始しなくても、広告は配信されたものとしてカウントされます。
