    Brightcove ブプレーヤ 5 から 6 への移行ガイド

    このトピックでは、BrightcovePlayerバージョン5からバージョン6に移行する際の問題について学習します。

    概要

    Video.js 6での後方互換性のない変更は、 Video.js wiki、このドキュメントは、Video.js5から6への移行には適用されないBrightcovePlayer6に固有の追加の移行ガイドラインに使用できます。

    Brightcove Player 5の重要な部分は、プレーヤー構成値によってトリガーされる互換性プラグインでした。

    "compatibility": true

    この構成は、5.xプレーヤーではデフォルトでオンになっています。その効果は、Video.js 4 APIを「シム」するプラグインを含めることです。これにより、Brightcove Player5への自動更新でより多くの顧客プラグインと統合がシームレスに機能できるようになりました。

    Brightcove Player 6以降、このシムはプレーヤーの一部ではなくなりました。5から6に更新したい人は、前述のドキュメントに従って、コードがVideo.js 6APIと互換性があることを確認する必要があります。 Video.js wiki。次のセクションでは、更新時に考慮する必要のある変更について説明します。

    player.tech

    Brightcove Playerのバージョン6では、player.techは現在の再生テクノロジーを返す関数であり、バージョン5のようなプレーヤーのプロパティではありません。シムはいくつかのプロパティをtechオブジェクトからtech関数にマップしたので、コードはplayer.tech技術オブジェクトになることはエラーをスローしません。

    プレーヤーのサイズ変更

    Video.js 4では、プレーヤーはJavaScriptを介してサイズ設定されていました。Video.js 5は、プレーヤーのサイズにいくつかの問題を引き起こす新しいCSSベースのサイズ設定スキームを導入しました。シムはこれらの変更をロールバックし、バージョン4の場合と同じように動作するようにしました。

    実際には、Brightcove Playerv6を使用する場合heightそしてwidthビデオタグのはピクセル単位で測定されますのみ。実際、あなたは何の単位も入力しませんheightそしてwidth属性：

    width="480"

    幅と高さを次のように設定できます100％バージョン5では、これはバージョン6では機能しなくなりました。これは、CSSを使用して2つの異なる方法で実現できます。

    • を使ってstyleの属性video鬼ごっこ。 
      <video-js data-video-id="5622718636001"
  data-account="1507807800001"
  data-player="default"
  data-embed="default"
  data-application-id
  class="video-js"
  controls
  style="width: 100%;height: 100%"></video-js>
<script src="https://players.brightcove.net/1507807800001/default_default/index.min.js"></script>
    • styleタグを使用する。 
        ...
  <style>
      .video-js {
        height: 100%;

  </style>
  ...
  <video-js data-account="1752604059001"

    コンポーネントコンストラクター

    Video.js 4、およびシムを介したバージョン5では、コンポーネントコンストラクター（PlayerそしてControlBar）のプロパティとして利用可能でしたvideojs関数：

     videojs.${ComponentName}

    これはもはやそうではありません。ザ・videojs.getComponent代わりに関数を使用する必要があります。

    var ControlBar = videojs.getComponent('ControlBar');

    SliderHandleコンポーネント

    このコンポーネントは5.xプレーヤーにはありませんでしたが、それを拡張していたユーザーのためにシムを介して復元されました。現在は完全に削除されています。

    プロパティ/方法と代替案

    以下は、互換性シムによって4.xから5.xにコピーされたVideo.jsプロパティの数です。これらは6.xには存在しなくなりましたが、代替手段が提供されています。

    削除済み 代替案
    vjs videojs
    videojs.JSON JSON
    videojs.USER_AGENT window.navigator.userAgent
    videojs.EventEmitter videojs.EventTarget
    videojs.obj.isArray Array.isArray
    videojs.round（num、digits） 数値（num.toFixed（digits））
    videojs.trim（str） str.trim（）
    videojs.util.mergeOptions videojs.mergeOptions

    カスタムプラグインの作成

    Brightcove Playerのバージョン5では、カスタムプラグインを作成したときに、プラグインの最初の行でvideojs.plugin()方法。例は次のとおりです。

    videojs.plugin('scrollIntoView', function() {
  var myPlayer = this,
    isAdPlaying = false;
  ...

    これで、videojs.registerPlugin()代わりにメソッド。例は次のとおりです。

    videojs.registerPlugin('scrollIntoView', function() {
  var myPlayer = this,
    isAdPlaying = false;

    でもvideojs.plugin()バージョン6では非推奨です。使用した場合、エラーは発生しませんが、コンソールに警告が表示されます。使用できませんvideojs.registerPlugin()バージョン5のプレーヤーで。

    両方のバージョンのプレーヤーでプラグインを使用する必要がある場合は、この手法を使用してVideo.js5 / 6の相互互換性を提供できます。 
    var registerPlugin =   videojs.registerPlugin || videojs.plugin;
registerPlugin('pluginName', function(){
  ...
});

    IMA3プラグイン

    Brightcove Player 6にアップグレードした後、Studioを使用してIMA3プラグインを手動で再インストールする必要があります。ステップバイステップを参照してください。広告の実装詳細についてはドキュメント。

    使用に慣れている場合プレーヤー管理API、curlを使用してIMA3プラグインをアップグレードすることもできます。具体的には、これがへのAPIリファレンスですプレーヤーを更新する。これは同じ方法で実行できることに注意してください 実際にプレーヤーをアップグレードするために使用するプレーヤー管理API呼び出し。

    FlashHLSとcrossdomain.xml

    Brightcove Playerリリース6.13.0以降を使用する特別な場合には、 crossdomain.xmlファイル。これは、次の場合に必要になります。

    • 独自のCDNを使用しています。
    • Flashを使用する必要があるような方法でBrightcovePlayerを使用しています。を参照してくださいどのレンディションが再生されるかを決定するこれが発生する可能性がある場合のドキュメント。

    これらの基準が満たされている場合は、 crossdomain.xmlメディアが提供されているルートドメインのファイル。たとえば、次のコンテンツを提供している場合

    https://www.domain.com/media/video.m3u8

    その後、 crossdomain.xmlファイルは上に存在する必要があります

    https://www.domain.com/crossdomain.xml

    プレーヤーswfはでホストされているのでplayers.brightcove.net、使用することをお勧めします*.brightcove.net。例crossdomain.xml次のように表示されます。

    <?xml version="1.0"?>
<cross-domain-policy>
<allow-access-from domain="*.brightcove.net" />
</cross-domain-policy>

    適切なドメインを指定する必要があります。詳細については、を参照してください。簡単なヒント：クロスドメインポリシーファイルのガイド。

    HTTPSでの作業

    HTTPページからHTTPSアセットをロードする場合、またはその逆の場合は、secureの値を持つ属性false次のように：

    <?xml version="1.0"?>
<cross-domain-policy>
<allow-access-from domain="*.brightcove.net" secure="false"/>
</cross-domain-policy>
