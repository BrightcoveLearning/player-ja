    プレーヤーカタログ

    このトピックでは、BrightcovePlayerの使用方法を学習します。カタログ VideoCloudカタログにリクエストを送信するためのメソッドのライブラリ。カタログのライブラリを使用すると、Video Cloudメディアに関する情報を簡単に取得して、プレーヤーにロードできます。

    このドキュメントのコンテンツは Video Cloud にのみ適用されます。

    基本構文

    を使用するための基本的な構文catalog以下のとおりであります：

    • 方法：playerName.catalog.methodName()

    のいくつかcatalogメソッドは、リクエストを定義するパラメータのオブジェクトを渡します。ザ・catalogパラメータオブジェクトは次のセクションで定義され、該当する場合はメソッドの説明で参照されます。

    注：このページのすべてのコンテンツで、パラメータが角かっこで囲まれている場合（[ ]）パラメータがオプションであることを示します。

    catalog parameters object

    次の表で、CatalogParamsオブジェクト。この真のオブジェクト（コンストラクターではない）は、多くのカタログ要求で使用されるパラメーターの形式を定義します。

    名前 タイプ 説明
    type string

    行うリクエストのタイプ。のいずれかである必要がありますビデオ、プレイリスト、または探す。
    policyKey string

    このアカウントのポリシーキー。これはデフォルトでプレーヤーのポリシーキーになります。** Note that if you pass a different account ID as a parameter, you must also pass the corresponding policy key for that account.
    deliveryConfigId string

    配信ルールID。見る概要：配達規則。
    [id] string

    接頭辞「ref：」が付いた動画または再生リストIDまたは参照ID。Requiredビデオとプレイリストのリクエスト用。DEPRECATED: For backward-compatibility, this is supported as a search query instead of q.これはサブオブジェクトとして提供できますが、これはCatalogParamsオブジェクト。パラメータを提供するために使用できます。
    [q] string

    検索クエリ。Required検索リクエストの場合、他の場合は無視されます。
    [adConfigId] string

    ビデオはSSAI広告構成IDである可能性があります。
    [tveToken] string

    クエリ文字列パラメータとして送信されるオプションのTVEトークン。
    [limit] string

    playlistsearchおよびタイプでのみサポートされます。返される動画の数を制限します。
    [offset] string

    playlistsearchおよびタイプでのみサポートされます。スキップする動画の数。
    [sort] string

    サポートされているsearchタイプのみ。検索のために動画を並べ替える方法。
    [bcovAuthToken] string Brightcove再生認証サービス（PAS）で使用します。

    **Note:デフォルトでは、プレーヤーで生成されたポリシーキーは検索が有効になっていません。つまり、次のいずれかを行う必要があります（いずれかを選択してください）。

    • 使用policyKey検索対応のポリシーキーを渡すパラメータ。

    • 使用プレーヤー管理APIプレーヤーのデフォルトのポリシーキーを上書きします。これを行うには、次のようなcurlステートメントを使用できます。

      curl \
  --header "Content-Type: application/json" \
  --user $EMAIL \
  --request PATCH \
  --data '{
    "video_cloud": {
      "policy_key": "YOUR SEARCH ENABLED POLICY KEY"
    }
  }' \
https://players.api.brightcove.com/v2/accounts/$ACCOUNT_ID/players/$PLAYER_ID/configuration

      検索対応のポリシーキーをプレーヤーに関連付けるときは、セキュリティの問題を必ず考慮してください。

      検索対応のポリシーキーの作成については、概要：ポリシーAPI-ビデオの検索セクション。

    getVideo( ) method

    ザ・getVideo(params | videoID,callback,[adConfigId]) => XMLHttpRequestメソッドはcatalog指定されたIDの動画をリクエストし、リクエストが完了するとコールバックを呼び出します。

    パラメーター

    • params | videoID:タイプのオブジェクトcatalogパラメーターまたは、プレフィックスが付いたビデオIDまたは参照IDを含む文字列ref:。

    • callback:リクエストが成功したかどうかにかかわらず、リクエストが完了したときに呼び出される関数。これは、2つの引数で呼び出されます。

      • エラーが発生した場合、最初の引数には Error オブジェクトが含まれます。このオブジェクトには、エラーの詳細が含まれています。エラーが発生しなかった場合、このオブジェクトはですnull。

      • エラーが発生しない場合、2 番目の引数は要求されたビデオオブジェクトです。エラーが発生した場合、この値には空の文字列が含まれます。

    • [adConfigId]:非推奨 Playback API に渡される Video Cloud 広告設定 ID。代わりに、最初のパラメータオブジェクトに adConfigId を渡します。

    戻り値

    catalog呼び出し自体は、型のオブジェクトを返しますXMLHttpRequest。このオブジェクトの例は次のとおりです。

    XMLHttpRequest呼び出しはオブジェクトを返しますが、ほとんどの場合、そのオブジェクトを処理することはありません。その理由は、コールバック関数ではJSON応答が自動的に解析され、使用可能なオブジェクトがコールバック関数に渡されるもの（この場合はビデオオブジェクト）であるためです。

    例:文字列としてのビデオID

    次の例は、getVideo()方法：

    <video-js id="myPlayerID"
  data-account="1507807800001"
  data-player="3bc25a34-086a-4060-8745-dd87af3d53b4"
  data-embed="default"
  data-video-id=""
  class="video-js" controls></video-js>
<script src="https://brightcove.net/1507807800001/3bc25a34-086a-4060-8745-dd87af3d53b4_default/index.min.js"></script>


<script type="text/javascript">
  videojs.getPlayer('myPlayerID').ready(function() {
    var myPlayer = this;
    myPlayer.catalog.getVideo('2114345471001', function(error, video){
    //deal with error
    myPlayer.catalog.load(video);
  });
});
</script>

    例:catalogParamsの使用

    上記のコードと同等ですが、CatalogParamsオブジェクトを使用すると次のようになります。

    <script>
  videojs.getPlayer('myPlayerID').ready(function() {
    var myPlayer = this,
      catalogParams = {};
    catalogParams.type = 'video';
    catalogParams.id = '2114345471001';
    myPlayer.catalog.getVideo(catalogParams, function(error, video){
    //deal with error
    myPlayer.catalog.load(video);
  });
});
</script>

    getPlaylist( ) method

    ザ・getPlaylist(params | playlistID,callback,[adConfigId]) => XMLHttpRequestメソッドはcatalog指定されたIDのプレイリストをリクエストし、リクエストが完了するとコールバックを呼び出します。

    パラメーター

    • params | playlistID:タイプのオブジェクトcatalogパラメーターまたは、プレイリストIDまたは参照IDをプレフィックスとして含む文字列ref:。

    • callback:リクエストが成功したかどうかにかかわらず、リクエストが完了したときに呼び出される関数。これは、2つの引数で呼び出されます。

      • エラーが発生した場合、最初の引数には Error オブジェクトが含まれます。このオブジェクトには、エラーの詳細が含まれています。エラーが発生しなかった場合、このオブジェクトはですnull。

      • エラーが発生しない場合、2 番目の引数は要求されたビデオオブジェクトです。エラーが発生した場合、この値には空の文字列が含まれます。

    • [adConfigId]:非推奨 Playback API に渡される Video Cloud 広告設定 ID。代わりに、最初のパラメータオブジェクトに adConfigId を渡します。

    戻り値

    ザ・catalog呼び出し自体はXMLHttpRequest。XMLHttpRequest呼び出しはオブジェクトを返しますが、ほとんどの場合、そのオブジェクトを処理することはありません。その理由は、コールバック関数ではJSON応答が自動的に解析され、使用可能なオブジェクトがコールバック関数に渡されるものであるためです。この場合はプレイリストオブジェクトです。

    例

    次の例は、getPlaylist()方法は次のとおりです。

    <video-js id="myPlayerID"
  data-account="1752604059001"
  data-player="f50a2d3c-af51-4d8c-84e3-0c7cdec0edbe"
  data-embed="default"
  class="video-js" controls></video-js>

<script src="https://players.brightcove.net/1752604059001/f50a2d3c-af51-4d8c-84e3-0c7cdec0edbe_default/index.min.js"></script>

<script type="text/javascript">
  videojs.getPlayer('myPlayerID').ready(function() {
    var myPlayer = this;
    myPlayer.catalog.getPlaylist('1754200320001', function(error, playlist){
      //deal with error
      myPlayer.catalog.load(playlist);
    })
  });
</script>
    プレイリストをロードしたいが、プレイリストの最初のビデオがプレーヤーに配置されていない場合は、playlist()メソッドで、2番目のパラメータをに設定することができます-1。たとえば、次のコードは、プレイリストを取得してプレーヤーに配置しますが、最初のビデオをプレーヤーにロードしません。 
    myPlayer.catalog.getPlaylist(playlistID, function(error, myPlaylist) {
  // deal with error
  // +++ Place playlist in player, but -1 parameter does not load playlist video in player +++
  myPlayer.playlist(myPlaylist, -1);
});
    もちろん、ビデオを手動でロードする必要があります。ブライトコーブプレーヤーのサンプルをご覧ください。このテクニックの例については、プレイリストの前にビデオを再生します。

    load（）メソッド

    ザ・load(mediaobject)メソッドは、ビデオまたはプレイリストオブジェクトをプレーヤーにロードします。このメソッドは、プレーヤーのを更新しますmediainfoプロパティ、およびビデオソースとポスターを更新します。ほとんどの場合、次の呼び出しの結果を使用してこのメソッドを呼び出します。getVideo()またはgetPlaylist()。通知を希望する場合mediainfo更新されました、聞いてくださいloadstartイベント。

    パラメーター

    • mediaobject:ロードするビデオまたはプレイリストオブジェクト。このオブジェクトは、への呼び出しからの応答オブジェクトと同じ形式である必要がありますgetVideo()またはgetPlaylist()方法。

    例

    VIDEO_CLOUD_ERR_NOT_PLAYABLE VideoCloudビデオは再生できません。 このエラーのよくある原因には、ビデオIDが非アクティブ化されたとき、またはスケジュールされた可用性が期限切れになったときが含まれます。

    扱うbc-catalog-error

    通常のエラー処理が可能ですready()のセクションscriptブロックは問題を引き起こす可能性があります。たとえば、bc-catalog-errorプレイヤーが準備が整う前にイベントが送出され、ready()セクション内のエラーをリッスンすると、エラーを処理できなくなります。この問題は、地理フィルタリングを使用する場合、ビデオが公開されていない、ビデオがスケジュール範囲外の場合、または別のアカウントで発生する可能性があります。コードに問題がない場合もありますが、問題はブラウザに依存する可能性があるため、徹底的にテストしてください。

    これが実装で可能性がある場合、エラー処理コードはone()リッスンするイベント処理メソッドbc-catalog-error内部ではなく、別のコードブロック内ready()セクション。

    以下のコードサンプルでは、コードブロックがbc-catalog-error（62行目）、およびのコードブロックready（71行目）。

    カスタムエラーの使用

    特定の処理に最適なソリューションbc-catalog-errorおそらくカスタムエラーを使用しています。次のコードは、カスタムエラーを使用して、非アクティブなビデオの視聴者にビデオが再生されない理由を通知するプラグインを示しています。

    videojs.registerPlugin('inactiveErrorCheck', function(options) {
  var myPlayer = this;
  myPlayer.one('bc-catalog-error', function(){
    var specificError;
    myPlayer.errors({
      'errors': {
        'inactive-error': {
          'headline': options.inactiveMessage,
          'dismiss': false
        }
      }
    });
    if (typeof(myPlayer.catalog.error) !== 'undefined') {
      specificError = myPlayer.catalog.error.data[0];
      if (specificError !== 'undefined' & specificError.error_code == "VIDEO_NOT_PLAYABLE") {
        myPlayer.error({code:'inactive-error'});
      };
    };
  });
});
