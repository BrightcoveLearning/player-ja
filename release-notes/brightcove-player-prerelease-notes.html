1〜2週間ごとに、プレリリースバージョンが現在のバージョンになることに注意してください。その時点で、ロックされていないすべてのプレーヤーが現在のバージョンにアップグレードされます。何らかの理由で特定のバージョンでプレイヤーをロックしたい場合は、プレーヤーの自動更新のオプトアウト（外部サイト）を参照してください。
 
 

新しいバージョンを試す

プレリリースバージョンが利用可能な場合、プレリリース版を試す最も簡単な方法は、Studio の PLAYERS モジュールに移動して、変更するプレーヤーをクリックしてから、プレーヤー情報を編集することです。バージョンセレクターを使用して、最新のプレリリースを選択します。

curlステートメントを使用して、プレーヤーをプレリリースバージョンに更新できます。カールの簡単な概要は、ステップバイステップの上部にあります：プレーヤー管理 (外部サイト) 。ドキュメントには、プレーヤーを更新する方法に関する特定のセクションがあります。

プレーヤーを更新するcurlコマンドを次に示します。ここで、 YOUR_EMAIL、 YOUR_ACCOUNT_IDそしてYOUR_PLAYER_ID。 

curl \
  --header "Content-Type: application/json" \
  --user YOUR_EMAIL \
  --request PATCH \
  --data '{
    "player": {
    "template": {
      "version": "6.57.0"
      }
    }
  }' \
  https://players.api.brightcove.com/v2/accounts/YOUR_ACCOUNT_ID/players/YOUR_PLAYER_ID/configuration

ほとんどの場合、プレーヤーを公開する必要があります。これは、次のcurlステートメントで実行できます。ここでも、示された場所に正しい値を代入する必要があります。 

curl \
  --header "Content-Type: application/json" \
  --user YOUR_EMAIL  \
  --request POST \
  https://players.api.brightcove.com/v2/accounts/YOUR_ACCOUNT/players/YOUR_PLAYER_ID/publish
