    プレーヤーのサイズ変更

    このトピックでは、iframeとembed_in_pageの実装を使用してビデオとプレーヤーのサイズを設定する方法を示します。

    サイズに関する考慮事項

    ビデオとプレーヤーのサイズを決定する際に留意すべき2つの重要な概念があります。

    • アスペクト比–プレーヤーを作成するときにカールステートメントで設定されるメディアの幅と高さのプロパティは、アスペクト比とも呼ばれる固有のビデオサイズを定義します。

    • プレーヤーのサイズ– CSSまたはHTMLコードで設定されたプレーヤーの幅と高さのプロパティは、プレーヤーのサイズを制御します。プレーヤーのサイズを変更すると、固有のビデオサイズが使用され、コントロールが正しく配置されたままになります。

    アスペクト比を設定する

    ビデオのアスペクト比は、幅と高さの関係です。クラシックTVは4：3のアスペクト比を使用しますが、ワイドスクリーンは16：9のアスペクト比を使用します。

    プレーヤーを作成するときに、Studioを使用してビデオの幅と高さを指定できます。使用Playersモジュール、プレイヤー情報セクション。

    注：ビデオのアスペクト比を決定するには、高さを幅で割るだけです。たとえば、高さが360ピクセル、幅が640ピクセルのビデオの比率は、.5625（360/640 = .5625）、つまり16：9（9/16 = .5625）です。

    ビデオのアスペクト比は同じままで、プレーヤーのサイズが変更されると、プレーヤーのコントロールが正しく配置されます。

    間違った比率を使用する

    ビデオのサイズとは異なるメディアアスペクト比を指定すると、ビデオの周囲にレターボックスが表示され、プレーヤーコントロールはビデオの上に配置されません。

    ここでは、16：9のアスペクト比を持つTigerビデオを使用しながら、メディアの幅と高さをそれぞれ640pxと480px（4：3のアスペクト比）に設定します。

    正しい比率を使用する

    メディアの寸法がビデオのアスペクト比と一致する場合、レターボックスは表示されず、プレーヤーのサイズを変更してもプレーヤーコントロールは正しく配置されたままになります。

    ここでは、メディアの幅と高さをそれぞれ640pxと360px（アスペクト比16：9）に設定しました。これは、Tigerビデオと同じです。

    プレーヤーサイズを設定する

    Studioを使用してプレーヤーのサイズを設定できます。使用プレイヤーモジュール、プレイヤー情報セクション。

    次のセクションで詳しく説明するように、CSSを使用してプレーヤーのサイズを設定することもできます。

    iframeを使う

    iframe実装の場合、プレーヤーのサイズは、widthそしてheightの属性iframe素子。これを行うにはいくつかの方法があります。

    • iframe要素に属性を追加する

    • iframe要素でCSSを使用する

    • id属性で CSS を使用する

    注：By / styleing /もの-sizing-brightcove-player.html設定するwidthそしてheight属性を使用して、サイズが固定されたプレーヤーを作成します。応答性の高いサイジングプレーヤーを作成するには、 Brightcove Player ドキュメントの「レスポンシブサイジング」を参照してください。

    iframe要素に属性を追加する

    追加して設定しますwidthそしてheightに属性iframe素子。

    <!doctype html>
<html>

<head>
  <meta charset="UTF-8">
  <title>Size the Player</title>
</head>

<body>
  <iframe src='https://players.brightcove.net/1752604059001/0711ceb1-07cf-430c-bd6f-951937e2c488_default/index.html'
    allowfullscreen webkitallowfullscreen mozallowfullscreen
    width="640" height="360"></iframe>

</body>
</html>
    を使用する場合widthおよび/またはheight iframeまたはページ内埋め込みプレーヤーの実装のいずれかにインラインで属性があります。単位はピクセルです。含めることができますpx番号の後ですが、技術的には適切なHTML構文ではありません。を参照してくださいパーセンテージの使用パーセンテージ値の使用方法については、このドキュメントの後半のセクションを参照してください。

    フルスクリーンディスプレイ

    フルスクリーンAPIをサポートする新しいブラウザーでは、フルスクリーンに切り替えたときにプレーヤーが100%に拡大されるよう、ページ内CSSルールを適用する必要があります。そうしないと、プレーヤーはフルスクリーン表示内に元のサイズで表示されます。

    次のCSSを使用して、フルスクリーンモードでプレーヤーがスケーリングするようにします。

    /***************************
* Fullscreen display      *
***************************/
body.vjs-full-window {
  padding: 0;
  margin: 0;
  height: 100%;
}
.video-js.vjs-fullscreen {
  position: fixed;
  overflow: hidden;
  z-index: 1000;
  left: 0;

    優先順位

    プレーヤーのサイズを設定する方法はいくつかあります。以下は、優先順位のリストです。

    • ランタイムの変更（JavaScriptを使用してプレーヤーのサイズを変更します）
    • <video-js>タグで設定されたサイズ
    • CSS（通常の継承規則に従う）
    • プレーヤーの構成設定

    PlayerAPIのサイズ

    ピクセル単位でサイズを返すPlayerAPIには4つのプレーヤーメソッドがありますが、それらの違いを理解することが重要です。

    • .height() - returns the height set in the height attribute of the video tag (if that attribute is not set, height() will return 0)
    • .width() - returns the width set in the width attribute of the video tag (if that attribute is not set, width() will return 0)
    • .currentHeight() - returns the current height of the player, however it was set
    • .currentWidth() - returns the current width of the player, however it was set

    ほとんどの場合、currentHeight()そしてcurrentWidth()使用したい方法です。
