ブライトコーブは、すべてのブライトコーブのサービスとアプリケーションのステータスを継続的に監視します。サービスに何らかの中断があると、システムステータスページに表示されます。スケジュールされたメンテナンスの通知もすべてこのページに掲載されます。

システムステータスページにアクセスするには、 https://status.brightcove.comにアクセスするか、ナビゲーションヘッダーから [サポート] アイコン ()、 システムステータス。

投稿されたすべての通知には、次の情報が含まれます。

  • インシデントステータス
    • オペレーション (緑)
    • 通知（青-システムは現在動作中ですが、遅延または断続的な停止の影響を受ける）
    • 部分的なサービス中断 (黄色)
    • フルサービスの中断 (赤)
  • コンポーネント-サービスまたは影響を受ける領域
  • 所在地
    • APAC
    • 南北アメリカ
    • EMEA
    • 日本
  • 現在の状態とステータス。州には以下が含まれます。
    • 調査中-エンジニアが問題を調査している
    • 特定-エンジニアが問題を特定し、修正を検討している
    • 監視-問題が解決され、すべてが正常であることを確認するためのエンジニアリングモニタリング
    • 解決済み-問題が解決されました

投稿されたメッセージはすべて英語で、使用時間は東部標準時（GMT-5）です。

システムステータス履歴の表示

システムステータスの履歴を表示するには、ページ上部の[履歴]リンクをクリックします。

システムステータスの変更の購読

サインアップして、システムステータスが変更されたときに通知を受け取ることができます。このセクションは、次のように構成されています。

  • メールを購読する
  • Webhook を購読する
  • サブスクリプションを管理する

メールを購読する

電子メールで通知を受信するには、[ 購読]ボタンをクリックし、メールアドレスを入力し、[ 購読] をクリックします。

Webhook を購読する

Webhook は、HTTP POST で作成された定義されたコールバックです。ステータスが更新されたときにコールバックが JSON オブジェクトを投稿する URL を指定します。

システムステータスが変更されたときに通知を受け取るようにサインアップするには、[ SUBSCRIBE]ボタンを選択します。次に、メールアドレスとともに JSON ペイロードを受信する URL を入力します。

システムステータスの変更が送信されると、Webhook アドレスは次のような JSON オブジェクトを受け取る必要があります。

サブスクリプションを管理する

購読すると、サブスクリプションを管理できる [サブスクリプションの管理] ページへのリンクが記載されたメールが届きます。通知の受信を希望するコンポーネント/製品/リージョンを選択できます。項目を選択し、[購読の保存]をクリックします。

購読を解除するには、[サブスクリプションの管理] ページの下部にある [ 登録解除]リンクをクリックします。

Slack で通知を受け取る

Slack には、ステータス更新の接続に使用できる独自の Webhook システムがあります。詳細については、 Slack のドキュメントを参照してください。
